## 1. 1С:Стандарты разработки V8 (v8-code-style)

**Репозиторий:** https://github.com/1C-Company/v8-code-style
**Документация стандартов 1С:** https://its.1c.ru/db/v8std

Расширение для 1C:EDT (входит в дистрибутив EDT 2021.3+). Проверки по стандартам 1С.

### Проверки модулей (BSL) -- основные категории

- **Транзакции:** начало транзакции только внутри блока Попытка-Исключение; корректные КоммитТранзакции/ОтменаТранзакции.
- **Документирующие комментарии:** у экспортных процедур/функций -- секция "Описание", параметры с типами, возвращаемое значение для функций; описание оканчивается точкой; только дефис-минус в списках.
- **Структура модуля:** стандартные области верхнего уровня; методы в нужных регионах; переменные в разделе переменных; не более одной пустой строки подряд; пустые области и пустые методы не допускаются.
- **Имена:** правила имен переменных (PascalCase); необязательные параметры после обязательных; не слишком много параметров у метода; типизация параметров и возвращаемых значений в комментариях.
- **Формы и директивы:** в модуле формы -- директивы компиляции (&НаКлиенте, &НаСервере); нет экспортных методов в модуле команд формы в обход ограничений.
- **Общие модули:** хотя бы один экспортный метод; имя модуля по типу (Cached, Client, ServerCall и т.д.); без избыточного обращения по собственному имени.
- **Запросы:** не выполнять запрос в цикле; при необходимости -- алиасы полей в запросе.
- **Безопасность:** перед Выполнить/Вычислить -- включение безопасного режима; удаление временных файлов после использования.
- **Прочее:** не пустой блок Исключение; не использовать оператор Перейти; устаревшие процедуры -- в области "УстаревшиеПроцедурыИФункции"; использование РольДоступна вместо РольВНабореРолей (IsInRole).

Итого: 88 проверок модулей (полный список в docs/checks/bsl.md репозитория v8-code-style).

## 2. BSL Language Server

**Сайт:** https://1c-syntax.github.io/bsl-language-server/
**Диагностики:** https://1c-syntax.github.io/bsl-language-server/en/diagnostics/

Реализация Language Server Protocol для 1С (BSL). Около 180 диагностик: Code smell, Error, Vulnerability, Security Hotspot.

### Важные группы диагностик

- **Стандарт (STANDARD):** каноническое написание ключевых слов, алиасы в запросах, транзакции, кэширование публичных методов, структура модуля и регионов, директивы компиляции, имена общих модулей (Cached, Client, ServerCall, FullAccess, Global), описание параметров и возвращаемых значений, пустой блок Исключение, выполнение запроса в цикле, удаление временных файлов, безопасный режим при Выполнить/Вычислить.
- **SQL:** запросы в цикле, OR в WHERE/JOIN, JOIN с подзапросами и виртуальными таблицами, FULL OUTER JOIN, проверка NULL для полей из соединений, некорректное использование LIKE.
- **Производительность (PERFORMANCE):** запрос в цикле, тяжелые JOIN.
- **Сложность (BRAINOVERLOAD):** когнитивная и цикломатическая сложность, магические числа/даты, смешение латиницы и кириллицы в одном идентификаторе.
- **Безопасность:** выполнение внешнего кода на сервере, отключение безопасного режима, доступ к ФС, обращение к интернету.
- **Устаревшее (DEPRECATED):** ТекущаяДата(), Сообщить(), Find, устаревшие методы платформы 8.3.10/8.3.12/8.3.17.

Игнорирование диагностики в коде: `// BSLLS:DiagnosticKey-off` (и снова включение: `// BSLLS:DiagnosticKey-on`).

## Как применять в проекте

- При написании кода: соблюдать стиль и правила из SKILL.md и REFERENCE.md; учитывать типичные срабатывания v8-code-style и BSL LS.
- При ревью: мысленно проверять код на перечисленные категории; предлагать правки по стандартам 1С и рекомендациям BSL LS.
- В проекте может не быть запущенного EDT или BSL LS -- этот текст дает контекст для ручной проверки и для генерации кода "с первого раза" ближе к стандартам.

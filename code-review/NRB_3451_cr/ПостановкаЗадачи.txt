Введение
Доработать загрузчик данных с Рудата, чтобы при загрузке нового кода НРД эмитента текущие коды НРД закрывались.

Доработка загрузчика ценных бумаг и эмитентов с Рудата
Как есть
Сейчас код НРД ЮЛ эмитента берется из поля issuer_nrd метода v2/Emitent/List. Если из этого поля получено значение, которого нет в 1С, то создается новый код НРД для этого ЮЛ (справочник "Коды загрузки", реквизит "Биржа" = "NRD code"). В ином случае, код НРД остается без изменений.
Как надо
При загрузке с Рудата отобрать в 1С код НРД ЮЛ эмитента (справочник "Коды загрузки", реквизит "Биржа" = "NRD code"), у которого реквизит "Дата начала действия" заполнен датой не позже даты загрузки и реквизит "Дата окончания действия" не заполнен, либо реквизит "Дата окончания действия" заполнен датой позже даты загрузки, либо оба эти реквизита не заполнены.
Если код не найден, то создать код НРД для этого ЮЛ (поле issuer_nrd метод v2/Emitent/List), в реквизит "Дата начала действия" записать текущую дату, реквизит "Дата окончания действия" не заполнять.
Если найден один код, то:
Если его значение совпадает со значением из поля issuer_nrd, то очистить у него реквизит "Дата окончания действия", остальное не менять.
В противном случае, у найденного кода НРД в реквизите "Дата окончания действия" заполнить текущую дату -1 день, и создать код НРД для этого ЮЛ (поле issuer_nrd), в реквизит "Дата начала действия" записать текущую дату, реквизит "Дата окончания действия" не заполнять.
Если найдено более одного кода, то выполнить следующие действия:
Если среди найденных есть 2 и более кода, у которых совпадают значения, то у одного из кодов (у любого) очистить реквизит "Дата окончания действия", остальное не менять. У остальных таких кодов в реквизите "Дата окончания действия" заполнить текущую дату -1 день.
Для оставшихся кодов (из найденных) и кода, у которого был очищен реквизит "Дата окончания действия" (если был дубль кода), выполнить проверку:
Если среди этих кодов есть код, значение которого совпадает со значением из поля issuer_nrd, то очистить у него реквизит "Дата окончания действия", остальное не менять. По остальным этим кодам в реквизите "Дата окончания действия" заполнить текущую дату -1 день.
В противном случае, у всех этих кодов в реквизите "Дата окончания действия" заполнить текущую дату -1 день, и создать код НРД для этого ЮЛ (поле issuer_nrd), в реквизит "Дата начала действия" записать текущую дату, реквизит "Дата окончания действия" не заполнять.
Тестирование
Проверить загрузку кода НРД эмитентов с Рудата с учетом изменений выше.
Выполнить регрессионное тестирование загрузчика данных с Рудата. Проверить также, что загружаются новые бумаги/эмитенты, апдейты бумаг/эмитентов, нкд, признак МК, признак обращаемости.
Два предыдущих пункта протестировать для следующих вариантов загрузки:
Загрузка рег. заданием.
Загрузка обработкой.

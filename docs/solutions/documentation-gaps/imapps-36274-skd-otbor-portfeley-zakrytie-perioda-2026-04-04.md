h1. IMAPPS-36274 — настройка запуска регламентных операций по фильтру без РДУ

h2. Исходные данные

* *Задача в Jira:* IMAPPS-36274
* *База:* WIM_FIn
* *Расширение-референс:* FP
* *Компонент (по ТЗ):* AvancoreFinance
* *Основание документа:* техническое задание по фактической реализации (imapps36274_05)

h2. Проблема (из описания в Jira)

* При большом числе договоров РДУ обработка *«Групповое выполнение закрытия периода»* работает долго.
* Отбор только по типам клиентов и ручному списку неудобен.
* РДУ попадает в общий тип «Физические лица», из-за чего сложно исключить РДУ без громоздкого ручного выбора.

*Цель:* добавить гибкий пользовательский отбор портфелей к существующей логике закрытия периода, не ломая штатный механизм.

h2. Реализованное решение (кратко)

_В Jira 8.13 (wiki) нумерованный список задаётся строками, начинающимися с «# »; ниже для файла Markdown использована эквивалентная нумерация._

1. Пользователь задаёт отбор портфелей в элементе справочника {{НастройкиЗакрытияПортфелейПоРасписанию}} через СКД; настройки хранятся в реквизите {{НастройкиОтбораПортфелейСКД}} ({{ХранилищеЗначения}}).
2. В обработке {{ГрупповоеВыполнениеЗакрытияПериодов}} после штатного формирования списка портфелей выполняется дополнительная фильтрация по сохранённой СКД.
3. Если СКД-настройка отсутствует или пустая, поведение совпадает со штатным.

h2. Объекты и изменения

h3. Справочник {{НастройкиЗакрытияПортфелейПоРасписанию}}

* Реквизит: {{НастройкиОтбораПортфелейСКД}} ({{ХранилищеЗначения}}).
* Форма элемента: события с расширением FP (после типовых) — инициализация компоновщика по макету {{Справочники.Портфели.ПолучитьМакет("МакетОтбораПортфелей")}}, загрузка/сохранение настроек в указанный реквизит.

h3. Справочник {{Портфели}}

* В менеджере: экспортная функция {{ПолучитьПортфелиПоОтборуСКД(НастройкаЗакрытияПортфелейПоРасписанию, ИсходныйСписокПортфелей)}}; вспомогательная {{УстановитьПараметрыСКД(...)}}.
* Логика: при пустой настройке или пустом исходном списке возвращается исходный список; иначе выполняется макет СКД {{МакетОтбораПортфелей}} с параметром {{СписокПортфелей}}; результат — массив ссылок (колонка «Ссылка»).

h3. Обработка {{ГрупповоеВыполнениеЗакрытияПериодов}}

* Перехваты в модуле объекта FP: перенос ссылки настройки в реквизит обработки; после штатного получения действующих портфелей — вызов {{ПолучитьПортфелиПоОтборуСКД}} во всех ключевых точках заполнения списка/периодов (в т.ч. варианты по границам и ДНГ/ДКР).

h2. Нефункциональные ограничения

* Не менять базовый алгоритм штатного отбора — только пост-фильтрация.
* Параметры СКД, без небезопасной подстановки в текст запроса.
* Обратная совместимость с существующими настройками расписания.

h2. Критерии приёмки (для проверки)

1. После записи элемента настроек и повторного открытия отбор СКД восстанавливается.
2. При заполненной СКД список портфелей в обработке сужается согласно отбору.
3. При пустом {{НастройкиОтбораПортфелейСКД}} результат как до доработки.
4. Фильтр действует при интерактивном запуске и при регламентном по расписанию.
5. Нет ошибок в типовом сценарии закрытия периода.

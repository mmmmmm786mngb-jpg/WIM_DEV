Назначение обработки

Обработка выполняет пакетное исполнение сделок и плановых позиций в контуре ПИФ: T+n, РЕПО и FX. Поддерживает запуск в пользовательском и фоновом режиме, разные сценарии запуска (основной, утренний, валютный), отправку уведомлений и дополнительное групповое перепроведение документов через внешнюю обработку.

Технология и контекст выполнения

  • Интеграция в БСП через СведенияОВнешнейОбработке() и ВыполнитьКоманду().
  • Основные бизнес-операции делегируются в БизнесПроцессы.СделкиTn, БизнесПроцессы.СделкиРЕПО, БизнесПроцессы.БиржеваяСделкаСВалютой.
  • Для фона используется ВТБ_МодульФоновыхЗаданий и встроенные фоновые задания.
  • Коммуникации: журнал регистрации + почтовые уведомления через РаботаСПочтовымиСообщениями.
  • Настройки читаются из хранилища настроек дополнительной обработки.

Статистика и ключевые сценарии

Поддерживаемые команды фонового запуска:

  • ФоновоеИсполнениеСделок - основной запуск (T0).
  • ФоновоеИсполнениеСделокУтреннее - утренний запуск (T-1), без группового перепроведения.
  • ФоновоеИсполнениеВалютныеСделки - валютные сделки (T-1), с перепроведением.

Ключевые блоки исполнения: плановые позиции T+n, сделки РЕПО, сделки FX, опционально перепроведение документов.

Диаграммы работы

1) Основной поток выполнения

Старт команды
  -> ВыполнитьКоманду()
     -> ФоноваяОбработка(ИдентификаторКоманды)
        -> ПрочитатьНастройки()
        -> Установить режим (T0/T-1, валютные, перепроведение)
        -> ВыполнитьСделки(ФоновоеИсполнение=Истина)
           -> Исполнение плановых позиций T+n
           -> Исполнение сделок РЕПО
           -> Исполнение FX
           -> Формирование комментария и статусов
           -> Почтовая рассылка + журнал регистрации
           -> (опционально) запуск группового перепроведения
  -> Завершение

2) Поток группового перепроведения

Условие: Нет ошибок + включен флаг перепроведения
  -> Получить внешнюю обработку перепроведения
  -> Подготовить период (ДатаНачала/ДатаОкончания)
  -> Передать список фондов и параметры
  -> ПерепровестиДокументыМногопоточно(...)
  -> Сохранить лог (txt -> временное хранилище -> вложение)
  -> Отправить письмо о результате
  -> Записать события в журнал регистрации

Детали по модулю формы

Ограничение исходников: в выгрузке присутствуют формы ФормаОбычная и ФормаНастроек типа Ordinary, но их исполняемые модули хранятся в бинарном виде (Form.bin).

Из текущих XML-файлов можно подтвердить состав форм, но нельзя извлечь список процедур/функций модуля формы без дополнительной декомпиляции EPF.

  • Форма по умолчанию: ФормаОбычная.
  • Дополнительная форма: ФормаНастроек.
  • По структуре сценариев ожидается UI для выбора даты/фондов/режимов и ручного запуска процедур объекта.

Возможности и эксплуатационные эффекты

  • OK: автоматизация нескольких контуров в одном запуске и единая отчетность по результату.
  • INFO: гибкая маршрутизация запуска (основной, утренний, валютный), включая ожидание завершения активного фонового задания.
  • INFO: детальная трассировка через журнал регистрации с раздельными событиями по T+n, РЕПО, FX и перепроведению.
  • ERROR: риски связаны с внешними зависимостями (почта, фоновые задания, внешняя обработка перепроведения).

Выводы

Обработка выступает как оркестратор регламентного контура исполнения и пост-обработки. Основная ценность - консолидация запуска T+n, РЕПО и FX в единой точке с унифицированным контролем, журналированием и рассылкой. Для полного документирования клиентской логики форм требуется отдельный шаг извлечения модулей из Form.bin.

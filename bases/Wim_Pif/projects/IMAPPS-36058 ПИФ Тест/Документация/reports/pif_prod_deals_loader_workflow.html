Назначение обработки

Обработка загружает сделки из внешнего сервиса (PROD) в контур ПИФ, подготавливает внутреннюю таблицу загрузки, рассчитывает комиссии, определяет торговые коды и создает целевые бизнес-процессы: внебиржевые сделки, РЕПО, сделки с валютой и связанные сущности.

Технология выполнения

  • Подключение к web-service через WSОпределения, WSПрокси, XDTO-фабрику.
  • Получение массива сделок вызовом GetDeals в ОтветЧерезСервис().
  • Промежуточная нормализация в табличную часть СписокЗагрузки.
  • Массовое создание объектов в СоздатьОбъекты() с учетом типа сделки и режима торгов.
  • Расчет комиссии по алгоритмам договоров в РассчитатьКомиссиюБрокераПоАлгоритмам().
  • Фоновый сценарий: задача интеграции, лог в XLS/ZIP, почтовая рассылка результата.

Статистика и рабочие команды

Команды, регистрируемые обработкой:

  • Пользовательский запуск загрузки через web-service.
  • Фоновая загрузка биржевых, внебиржевых, FX.
  • Отдельные режимы рублевого T0 и валютного T-1 сценария.

Счетчики в обработке: ВсегоПрочтено, ВсегоНовых, ВсегоЗагружено, ПрочитаноИзСделкиQuik.

Диаграммы работы

1) Сквозной конвейер загрузки

Команда запуска
  -> ВыполнитьКоманду()
     -> УстановитьНастройки()
     -> ЗагрузитьОбъекты(ВалютныеСделки, Отказ, ТекстОшибки)
        -> ОтветЧерезСервис(GetDeals)
        -> ПрочитатьНаСервере(Сделки, ВалютныеСделки, ...)
           -> Фильтрация и нормализация в СписокЗагрузки
           -> Расчет комиссий, определение режимов/торговых кодов/курсов
        -> СоздатьОбъекты(Отказ, ТекстОшибки)
           -> Создание БП/документов по типу сделки
           -> Проверки обязательных реквизитов
     -> Формирование лога и рассылки
  -> Завершение задачи интеграции

2) Ветвление по типам сделок при создании объектов

Строка СписокЗагрузки
  -> Определение режима (OTC / FX / REPO / прочее)
     -> OTC/MPTR/OCBR: БП ВнебиржеваяСделка
     -> FX: БП БиржеваяСделкаСВалютой
     -> REPO: БП СделкиРЕПО
     -> Прочее: биржевой сценарий T+n
  -> Заполнение реквизитов
  -> Проверка обязательных полей
  -> Запись + проведение/шаги БП
  -> Логирование результата

Ключевые детали и контрольные точки

  • INFO: фильтрация сделок зависит от режима загрузки и признака валютного сценария.
  • INFO: отдельный контроль по отсутствующим партнерам (код брокера) с адресной рассылкой предупреждений.
  • OK: загрузка работает пакетно, применяет кэширование и отборы для снижения нагрузки.
  • ERROR: критичны внешние зависимости: web-service доступность, корректность сопоставлений, настройки алгоритмов комиссий.

Выводы

Обработка реализует полный производственный конвейер интеграции сделок из PROD: от получения данных и интеллектуальной фильтрации до создания бизнес-процессов и пост-контроля. Ключевая особенность - богатое ветвление по типам сделок и настройкам режима, а также встроенный механизм фонового исполнения, протоколирования и уведомлений.

## Где что искать

### Путь к исходникам базы

- **Из папки базы** (например `bases/SBB_Broker/`): файл `source-path.txt` в этой папке.
- **Из проекта на одной базе** (`bases/<база>/projects/<проект>/`): `../../source-path.txt`.
- **Из shared-проекта**: для каждой базы — `../../<папка_базы>/source-path.txt` (например `../../SBB_Broker/source-path.txt`). Список баз можно хранить в `bases.txt` (по одной в строке).

### Структура одной базы

```
bases/<база>/
  source-path.txt   # одна строка — путь к каталогу в WORK
  README.md
  projects/
    <проект>/
      Расширения/
      Документация/
      Тестирование/
      Скрипты/
```

### Shared (несколько баз)

- **Папка:** `bases/shared/projects/`.
- **Шаблон:** `example/` (в т.ч. `bases.txt` — опционально список баз).
- **Пути к базам:** не копировать; читать из `../../<база>/source-path.txt`.

## Быстрый поиск по смыслу

| Нужно | Куда смотреть |
|-------|----------------|
| Имя папки базы по пути WORK | Таблица выше: столбец «Папка в репо» ↔ «Путь к исходникам» |
| Путь к исходникам по папке базы | `bases/<папка_базы>/source-path.txt` или таблица выше |
| **Назначение и состав конфигурации (что за база)** | **`bases/BASES_GLOSSARY.md`** — глоссарий по всем пяти базам |
| Проекты на конкретной базе | `bases/<база>/projects/` |
| Проекты по нескольким базам | `bases/shared/projects/` |
| Правила создания проектов | `.cursor/rules/project-structure.mdc`, `.cursor/rules/1c-project-conventions.mdc` |
| Структура репозитория | `bases/README.md` |

# Расширения

## Ext11111_WimDu

Расширение для конфигурации **Wim_Du** (путь к выгрузке: `bases/Wim_Du/source-path.txt` -> `C:\1c\Cursor_1c\WORK\Wim_Du`).

### Состав

- Отчет **Ext11111_ОстаткиДоговорДуПоСчету** (в интерфейсе тот же синоним «дополнительный отчет»): СКД, остатки `РегистрБухгалтерии.Хозрасчетный` на дату с разрезом по измерению **ДоговорДУ** и **Валюта** при отборе по выбранному **счету** (включая иерархию: `Счет В ИЕРАРХИИ`). Имя в метаданных сменено с `...ДУДоп`, чтобы обойти битые записи ИОМ в базе. Исходники: `Reports/Ext11111_ОстаткиДоговорДуПоСчету/`.
- Роль **Ext11111_ОсновнаяРоль** (права на отчет). В `Configuration.xml` секция **DefaultRoles** намеренно пустая: при `ConfigurationExtensionCompatibilityMode` 8.3.9 и режиме основной КД 8.3.9 заполненная **DefaultRoles** дает ошибку проверки метаданных про заимствованные объекты. Роль нужно назначать пользователям вручную.

### Параметры отчета

| Параметр | Назначение |
|----------|------------|
| Период   | Стандартный период; остатки берутся на **конец периода** (`ДатаОкончания` в запросе) |
| Счет     | Счет учета из плана счетов Хозрасчетный |

На форме отчета параметры **Период** и **Счет** выводятся в панель быстрых настроек СКД (`viewMode` = QuickAccess).

### Подключение

1. Загрузка из XML (пример): скрипт `db-load-xml.ps1` с `-InfoBaseServer`, `-InfoBaseRef`, `-ConfigDir` (каталог расширения), `-Extension "Ext11111_WimDu"`, при необходимости `-UpdateDB`.
2. Назначить пользователям роль **Ext11111_ОсновнаяРоль** (или добавить права на отчет в свои роли).

### Исходник определения СКД

Файл `Ext11111_WimDu/_skd_ostatki.json` можно править и пересобирать `Reports/Ext11111_ОстаткиДоговорДуПоСчету/.../Template.xml` скриптом `skd-compile` из репозитория навыков.

### Загрузка на сервер localhost / WIM_DU

Если при отладке создавалось расширение **Ext11111_WimDuSrv**, его можно удалить в конфигураторе (остается рабочее **Ext11111_WimDu**).

### Ошибка «не найден идентификатор» в справочнике ИОМ

После первой установки или обновления расширения запустите **1С:Предприятие** с параметром  
`/C "ЗапуститьОбновлениеИнформационнойБазы"`  
либо обновите данные справочника **Идентификаторы объектов метаданных** (см. `Документация/troubleshooting_metadata_identifiers.md`).

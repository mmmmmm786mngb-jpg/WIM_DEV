# Глоссарий базовых конфигураций WIM_DEV

Краткий справочник по составу и назначению каждой базы (для быстрого поиска при работе с кодом и проектами). Исходники конфигураций лежат в **C:\1c\Cursor_1c\WORK\<имя_папки>**.

## SBB_Broker

| Поле | Значение |
|------|----------|
| **Имя конфигурации** | ИнвестиционныйЦентр |
| **Синоним** | Backoffice RELEASE |
| **Вендор** | ООО CБ-Брокер |
| **Версия (на момент анализа)** | 2026.02.06.01 |
| **Расположение в WORK** | Корень: `Configuration.xml`, `ConfigDumpInfo.xml`, папки метаданных (Tasks, Catalogs, Documents и т.д.) |

**Ключевые объекты (по структуре):** Задачи (Tasks): ЭтапЗагрузкиБиржевыхОтчетов, РегистрацияКлиента, ЭтапФормированияСПД, ЭтапОткрытияДня, ЭтапОперацииСПаями, ЭтапВыгрузкаДанныхFlextera, ЭтапРегистрацииСчетовДляВозврата и др.

**Связь с другими базами:** Обмен с Депозитарием (SBB_Depo), бэк-офис для брокерских операций.

## Где смотреть в коде

- **Имя/синоним/вендор конфигурации:** в корне выгрузки файл `Configuration.xml` (или `SRC\CF\Configuration.xml`, `src\cf\Configuration.xml`), секция `<Properties>`: `<Name>`, `<Synonym>`, `<Vendor>`, `<Version>`.
- **Структура метаданных:** те же файлы — секция `<ChildObjects>` (Subsystem, Catalog, Document, Task, BusinessProcess и т.д.).
- **Различие выгрузок:** SBB_* — выгрузка в корень каталога; Wim_Du, Wim_Mo — подкаталог `SRC\CF`; WIM_FIn — подкаталог `src\cf` (основная конфигурация) и другие в `src\`.

Этот глоссарий можно использовать при поиске нужной базы по назначению или при уточнении состава конфигурации.

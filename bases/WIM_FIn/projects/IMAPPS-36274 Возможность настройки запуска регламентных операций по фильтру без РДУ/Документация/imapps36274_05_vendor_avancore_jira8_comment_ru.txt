h2. Задание для вендора: Аванкор (перенос доработки IMAPPS-36274)

*Задача Jira:* IMAPPS-36274
*Наименование:* Возможность настройки запуска регламентных операций по фильтру без РДУ
*Компонент:* AvancoreFinance
*База:* WIM_FIn
*Референс-расширение:* FP (фактическая реализация; документ-основа: imapps36274_05_fp_based_technical_spec)

----

h3. 1. Контекст и проблема

* При большом числе РДУ-договоров обработка {{Групповое выполнение закрытия периода}} работает долго.
* Отбор только по типам клиентов и ручному списку неудобен.
* РДУ попадает в общий тип «Физические лица», из-за чего сложно исключить РДУ без громоздкого ручного выбора.

*Цель:* добавить гибкий пользовательский отбор портфелей в существующую логику закрытия периода *без изменения* базового штатного отбора (только пост-фильтрация).

----

h3. 2. Ожидаемое поведение (функционально)

# Пользователь настраивает отбор портфелей в элементе справочника {{НастройкиЗакрытияПортфелейПоРасписанию}} через СКД.
# Настройки отбора сохраняются в реквизите типа {{ХранилищеЗначения}}.
# В обработке {{ГрупповоеВыполнениеЗакрытияПериодов}} сначала выполняется штатный отбор портфелей, затем применяется дополнительный фильтр по сохраненной СКД.
# Если СКД-настройка отсутствует или пустая, поведение полностью совпадает с текущим (без регрессии).
# Работает при интерактивном запуске и при запуске по расписанию (регламент).

----

h3. 3. Объекты метаданных и суть изменений

h4. Справочник {{НастройкиЗакрытияПортфелейПоРасписанию}}

* Добавить реквизит {{НастройкиОтбораПортфелейСКД}} (тип {{ХранилищеЗначения}} / {{v8:ValueStorage}}).
* В форме элемента:
** подключить события: {{OnCreateAtServer}} (callType={{After}}) -> {{fp_ПриСозданииНаСервереПосле}}; {{BeforeWriteAtServer}} (callType={{After}}) -> {{fp_ПередЗаписьюНаСервереПосле}};
** в модуле формы: инициализация компоновщика по макету {{Справочники.Портфели.ПолучитьМакет("МакетОтбораПортфелей")}}, загрузка сохраненных настроек (или по умолчанию), сохранение настроек компоновщика в реквизит {{НастройкиОтбораПортфелейСКД}}.

h4. Справочник {{Портфели}}

* В модуле менеджера:
** экспортная функция {{ПолучитьПортфелиПоОтборуСКД(НастройкаЗакрытияПортфелейПоРасписанию, ИсходныйСписокПортфелей)}};
** процедура {{УстановитьПараметрыСКД(Настройки, ИмяПараметра, Значение)}}.
* Логика {{ПолучитьПортфелиПоОтборуСКД}}: при отсутствии настройки или исходного списка вернуть исходный результат; иначе взять настройки из {{НастройкаЗакрытияПортфелейПоРасписанию.НастройкиОтбораПортфелейСКД}}, выполнить макет СКД {{МакетОтбораПортфелей}}, в параметр СКД {{СписокПортфелей}} передать исходный список, результат — массив ссылок (например {{ВыгрузитьКолонку("Ссылка")}}).
* Макет СКД: {{МакетОтбораПортфелей}}; в запросе использовать параметр {{СписокПортфелей}} для ограничения выборки.

h4. Обработка {{ГрупповоеВыполнениеЗакрытияПериодов}} (модуль объекта)

* {{&Вместо("ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия")}}: перенос ссылки настройки {{ЭтотОбъект.НастройкаЗакрытияПортфелейПоРасписанию = НастройкаРегулярногоЗакрытия}}.
* {{&ИзменениеИКонтроль("ЗаполнитьПериодыПортфелейПоГраницам")}}: после {{ДействующиеПортфели(...)}} вызвать фильтр {{Справочники.Портфели.ПолучитьПортфелиПоОтборуСКД(...)}}.
* {{&ИзменениеИКонтроль("ЗаполнитьПериодыПортфелейПоВарианту_ДНГ_ДКР")}}: аналогично.
* {{&Вместо("ЗаполнитьСписокПортфелей")}}: после получения {{ДействующиеПортфели}} вызвать {{ПолучитьПортфелиПоОтборуСКД(...)}}.

*Итог:* фильтрация СКД встроена во все ключевые места формирования набора портфелей для закрытия.

h3. 6. Критерии приемки

# После записи и повторного открытия элемента справочника отбор СКД сохранен.
# При заданной СКД список портфелей в обработке сужается согласно отбору.
# При пустом {{НастройкиОтбораПортфелейСКД}} результат совпадает со штатным поведением.
# При регламентном запуске по настройке расписания фильтр применяется.
# В типовом сценарии закрытия периода нет ошибок выполнения.

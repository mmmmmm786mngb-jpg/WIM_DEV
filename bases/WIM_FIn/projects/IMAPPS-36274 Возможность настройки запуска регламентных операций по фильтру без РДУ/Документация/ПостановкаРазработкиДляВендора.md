# Постановка задачи на доработку для вендора

**Задача:** IMAPPS-36274 Возможность настройки запуска регламентных операций по фильтру без РДУ  
**База:** WIM_FIn (ДУ редакция 2, Аванкор)  
**Дата:** 03.04.2026

## 1. Цель доработки

Ускорить и упростить отбор портфелей для группового закрытия периода за счет пользовательского отбора СКД, который применяется поверх уже существующего механизма формирования списка портфелей.

Ключевое требование:
- новый механизм **не заменяет** старый;
- функция фильтра принимает готовый список из старого механизма;
- если настройки СКД пустые, возвращается исходный список без изменений.

## 2. Фактическая схема в текущем WIM_FIn

### 2.1. Передача настройки из справочника в обработку

В `Catalogs.НастройкиЗакрытияПортфелейПоРасписанию.Ext.ObjectModule`:
- `ПередЗаписью()` создает/обновляет регламентное задание `ГрупповоеВыполнениеЗакрытияПериодаПоНастройкам`;
- `ПриЗаписи()` передает в задание параметр `Ссылка` на элемент настройки:
  - `ПараметрыЗадания = Новый Массив;`
  - `ПараметрыЗадания.Добавить(Ссылка);`

В `DataProcessors.ГрупповоеВыполнениеЗакрытияПериодов.Ext.ObjectModule`:
- `ВыполнитьРегулярноеРасписаниеЗакрытияПериода(НастройкаРегулярногоЗакрытия)` принимает ссылку настройки;
- `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия()` переносит значения в реквизиты обработки;
- `ЗаполнитьПериодыПортфелейПоНастройкеРегулярногоЗакрытия()` формирует периоды.

### 2.2. Где формируется список портфелей сейчас

В `DataProcessors.ГрупповоеВыполнениеЗакрытияПериодов.Ext.ObjectModule`:
- `ЗаполнитьСписокПортфелей(ДатаЗакрытия, СписокТипов, СписокПортфелейПоСтратегиям)` вызывает:
  - `Справочники.Портфели.ДействующиеПортфели(ДатаЗакрытия, СписокТипов, СписокПортфелейПоСтратегиям)`.

В `Catalogs.Портфели.Ext.ManagerModule`:
- `ДействующиеПортфели(ДатаСреза, ТипКлиентаДУ, СписокПортфелей)` выполняет запрос и возвращает массив ссылок;
- уже есть СКД-паттерн `СписокПортфелейПоДинамическомуСписку(...)` с:
  - `ПолучитьМакет("ДействующиеПортфели")`;
  - инициализацией `КомпоновщикНастроекКомпоновкиДанных`;
  - выполнением через `ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений`.

## 3. Референс по СКД (демо конфигурация)

В **демо конфигурации** (внутренний эталон, не поставка) используется рабочая схема справочника с пользовательским отбором СКД (условное имя объекта в демо: каталог контроля с макетом отбора и `НастройкиСКД` в `ХранилищеЗначения`):
- реквизит объекта `НастройкиСКД` типа `ХранилищеЗначения` (`v8:ValueStorage`);
- реквизит формы `КомпоновщикСхемы` типа `КомпоновщикНастроекКомпоновкиДанных`;
- в `Forms.ФормаЭлемента.Ext.Form.Module`:
  - `ПриСозданииНаСервере()`:
    - `СхемаКомпоновки = ДанныеОбъект.ПолучитьМакет("<ИмяМакетаОтбораВДемо>");`
    - `КомпоновщикСхемы.Инициализировать(...)`
    - `КомпоновщикСхемы.ЗагрузитьНастройки(...)` из `НастройкиСКД` или из `НастройкиПоУмолчанию`;
  - `ПередЗаписьюНаСервере()`:
    - `ТекущийОбъект.НастройкиСКД = Новый ХранилищеЗначения(ЭтотОбъект.КомпоновщикСхемы.Настройки);`
- в `Ext.ManagerModule`:
  - настройки из `НастройкиСКД.Получить()` исполняются через компоновщик/процессор.

Эта схема принимается как эталон хранения и исполнения пользовательского отбора.

## 4. Что добавить в WIM_FIn (EDT, не XML)

### 4.1. Новые реквизиты метаданных

Объект: `Справочник.НастройкиЗакрытияПортфелейПоРасписанию`

- `НастройкиОтбораПортфелейСКД`  
  Тип: `ХранилищеЗначения`  
  Назначение: хранение пользовательских настроек СКД.

### 4.2. Новые реквизиты формы

Форма: `Справочник.НастройкиЗакрытияПортфелейПоРасписанию.ФормаЭлемента`

- `КомпоновщикНастроекОтборПортфелей`  
  Тип: `КомпоновщикНастроекКомпоновкиДанных`.

Форма обработки: `Обработка.ГрупповоеВыполнениеЗакрытияПериодов.<рабочая форма>`

- `КомпоновщикНастроекОтборПортфелей`  
  Тип: `КомпоновщикНастроекКомпоновкиДанных`.

### 4.3. Новые элементы формы

На обеих формах:
- группа: `ГруппаОтборПортфелейСКД` (обычная группа);
- элемент: `КомпоновщикНастроекОтборПортфелейПоле`  
  Тип: `Таблица/Поле настроек компоновки данных`  
  Путь: `КомпоновщикНастроекОтборПортфелей.Настройки.Отбор`.

## 5. Изменения в макете СКД

Макет: `Справочники.Портфели.Темплейт.ДействующиеПортфели`

Добавить параметр:
- `СписокПортфелей` (тип коллекции ссылок, совместимый с `В (...)` в запросе).

Параметр используется для ограничения данных: СКД фильтрует только портфели, которые уже отобраны старым механизмом.

## 6. Точка встраивания в существующий код WIM_FIn

Вставка выполняется в `DataProcessors.ГрупповоеВыполнениеЗакрытияПериодов.Ext.ObjectModule`, в процедуре:
- `ЗаполнитьСписокПортфелей(...)`.

Текущий шаг:
1) Получается `ДействующиеПортфели` через `Справочники.Портфели.ДействующиеПортфели(...)`.

Новый шаг:
2) Этот результат передается в `ПолучитьПортфелиПоОтборуСКД(...)`.

Итог:
3) В `СписокПортфелей` обработки попадает отфильтрованный список.

## 7. Шаблоны кода

### 7.1. Форма элемента настройки: инициализация/сохранение

```bsl
// Описание: Инициализирует компоновщик отбора портфелей.
//
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
	
	СхемаКомпоновки = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");
	ИсточникНастроек = Новый ИсточникДоступныхНастроекКомпоновкиДанных(СхемаКомпоновки);
	КомпоновщикНастроекОтборПортфелей.Инициализировать(ИсточникНастроек);
	
	СохраненнаяНастройка = Объект.НастройкиОтбораПортфелейСКД.Получить();
	Если ЗначениеЗаполнено(СохраненнаяНастройка) Тогда
		КомпоновщикНастроекОтборПортфелей.ЗагрузитьНастройки(СохраненнаяНастройка);
	Иначе
		КомпоновщикНастроекОтборПортфелей.ЗагрузитьНастройки(СхемаКомпоновки.НастройкиПоУмолчанию);
	КонецЕсли;
	
КонецПроцедуры

&НаСервере
Процедура ПередЗаписьюНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи)
	
	ТекущийОбъект.НастройкиОтбораПортфелейСКД = Новый ХранилищеЗначения(
		КомпоновщикНастроекОтборПортфелей.Настройки);
	
КонецПроцедуры
```

### 7.2. Фильтрация списка портфелей по СКД

```bsl
// Описание: Фильтрует список портфелей настройками СКД.
// Параметры:
//  НастройкиОтбораСКД - ХранилищеЗначения - сохраненные настройки компоновщика
//  ИсходныйСписокПортфелей - Массив - список из старого механизма
//
// Возвращаемое значение:
//  Массив - отфильтрованный список
//
Функция ПолучитьПортфелиПоОтборуСКД(НастройкиОтбораСКД, ИсходныйСписокПортфелей) Экспорт
	
	Если Не ЗначениеЗаполнено(ИсходныйСписокПортфелей) Тогда
		Возврат Новый Массив;
	КонецЕсли;
	
	Если Не ЗначениеЗаполнено(НастройкиОтбораСКД) Тогда
		Возврат ИсходныйСписокПортфелей;
	КонецЕсли;
	
	НастройкиКомпоновки = НастройкиОтбораСКД.Получить();
	Если Не ЗначениеЗаполнено(НастройкиКомпоновки)
		Или НастройкиКомпоновки.Отбор.Элементы.Количество() = 0 Тогда
		Возврат ИсходныйСписокПортфелей;
	КонецЕсли;
	
	СхемаКомпоновки = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");
	ОбщегоНазначения.УстановитьПараметрыСКД(НастройкиКомпоновки, "СписокПортфелей", ИсходныйСписокПортфелей);
	
	ТаблицаЗначений = Новый ТаблицаЗначений;
	КомпоновщикМакета = Новый КомпоновщикМакетаКомпоновкиДанных;
	Макет = КомпоновщикМакета.Выполнить(
		СхемаКомпоновки,
		НастройкиКомпоновки,
		,
		,
		Тип("ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений"));
	
	Процессор = Новый ПроцессорКомпоновкиДанных;
	Процессор.Инициализировать(Макет);
	
	Вывод = Новый ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений;
	Вывод.УстановитьОбъект(ТаблицаЗначений);
	Вывод.Вывести(Процессор);
	
	Возврат ТаблицаЗначений.ВыгрузитьКолонку("Ссылка");
	
КонецФункции
```

### 7.3. Вызов в текущей процедуре заполнения

```bsl
// Фрагмент процедуры ЗаполнитьСписокПортфелей(...)
ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(
	ДатаЗакрытия,
	СписокТипов,
	СписокПортфелейПоСтратегиям);

ДействующиеПортфели = ПолучитьПортфелиПоОтборуСКД(
	НастройкаРегулярногоЗакрытия.НастройкиОтбораПортфелейСКД,
	ДействующиеПортфели);
```

## 8. Критерии приемки

1. Старый механизм отбора (типы клиентов + стратегии + список) работает без изменений.
2. При заполненных настройках СКД список портфелей дополнительно сужается.
3. При пустых настройках СКД результат идентичен текущему (без регресса).
4. Выполнение задания по расписанию использует тот же отбор, что и интерактивный запуск.

## 9. Что заимствовать в расширение для демо вендору

Ниже перечень объектов, которые рекомендуется заимствовать в расширение, чтобы показать доработку без изменения основной конфигурации.

   - реквизит: `НастройкиОтбораПортфелейСКД` (тип `ХранилищеЗначения`);
   - форма: `ФормаЭлемента` (добавить компоновщик и элемент отбора);
   - модуль формы: инициализация/сохранение настроек СКД.

2. `Обработка.ГрупповоеВыполнениеЗакрытияПериодов`
   - модуль объекта: добавить вызов фильтра в `ЗаполнитьСписокПортфелей(...)`;
   - рабочая форма (или формы): добавить реквизит формы `КомпоновщикНастроекОтборПортфелей` и элемент отбора.

3. `Справочник.Портфели`
   - макет `ДействующиеПортфели`: добавить параметр `СписокПортфелей` и условие использования параметра.

### 9.2. Расширенный контур (если нужно показать полный production-подход)

4. `Справочник.Портфели` модуль менеджера (`Ext.ManagerModule`)

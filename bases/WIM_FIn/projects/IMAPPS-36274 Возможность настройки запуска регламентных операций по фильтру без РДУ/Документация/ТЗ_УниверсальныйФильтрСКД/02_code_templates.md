# Готовые тексты BSL и структура формы (образец из демо конфигурации)

Перед вставкой проверить имена реквизитов и наличие параметров `ОграничитьКандидатами`, `КандидатыПортфели` в макете `ДействующиеПортфели` (см. `01_detailed_spec.md`, п. 6.1). Параметр даты дня в СКД **не используется** — кандидаты уже отобраны на нужную дату до вызова.

**Реквизит формы `КомпоновщикУниверсальногоФильтраПортфелейСКД`:** в метаданных тип **`КомпоновщикНастроекКомпоновкиДанных`**, в XML формы — **`dcsset:SettingsComposer`**. Таблица отбора: `КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter`.

---

## 0) Фрагменты `Form.xml` формы элемента справочника

### 0.1. Пространства имён в корневом элементе `Form`

Добавить (если отсутствуют) объявления:

- `xmlns:dcsset="http://v8.1c.ru/8.1/data-composition-system/settings"`

### 0.2. События формы

```xml
<Events>
    <Event name="OnCreateAtServer">ПриСозданииНаСервере</Event>
    <Event name="OnReadAtServer">ПриЧтенииНаСервере</Event>
    <Event name="BeforeWriteAtServer">ПередЗаписьюНаСервере</Event>
</Events>
```

Если обработчики объявлены в расширении с префиксом — подставить свои имена.

### 0.3. Реквизит формы (после `Объект`)

```xml
<Attribute name="КомпоновщикУниверсальногоФильтраПортфелейСКД" id="...">
    <Title>
        <v8:item>
            <v8:lang>ru</v8:lang>
            <v8:content>Универсальный фильтр СКД</v8:content>
        </v8:item>
    </Title>
    <Type>
        <v8:Type>dcsset:SettingsComposer</v8:Type>
    </Type>
</Attribute>
```

Скрытый реквизит формы (булево), см. п. 0.5:

```xml
<Attribute name="ФильтрУниверсальногоОтбораПортфелейСброшен" id="...">
    <Type><v8:Type>xs:boolean</v8:Type></Type>
</Attribute>
```

### 0.4. Таблица дерева отбора (по образцу демо конфигурации)

- `Representation` = `Tree`
- `DataPath` = `КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter`
- `InitialTreeView` = `ExpandAllLevels`
- `Autofill` = `true`

### 0.5. Команда «Очистить универсальный фильтр СКД»

Обычная команда формы + кнопка рядом с таблицей отбора. Отдельного признака «использовать фильтр» нет: незаполненное хранилище в объекте означает отсутствие дополнительного отбора СКД.

Для записи в БД значения `Неопределено` после очистки (при том что компоновщик снова содержит настройки по умолчанию) удобно завести **скрытый реквизит формы** `ФильтрУниверсальногоОтбораПортфелейСброшен` (Булево): команда очистки ставит `Истина`, в `ПередЗаписьюНаСервере` при `Истина` записывают в объект `Неопределено` и сбрасывают флаг, иначе сериализуют компоновщик.

## 1) `Catalogs/НастройкиЗакрытияПортфелейПоРасписанию/Ext/ObjectModule.bsl`

Опционально: вспомогательные процедуры чтения/записи хранилища (если выносите логику из формы). Отдельного реквизита с текстовым представлением отбора **нет**.

```bsl
// Описание: Сохраняет настройки универсального СКД-фильтра в реквизит элемента
//
// Параметры:
// НастройкиКомпоновки - НастройкиКомпоновкиДанных - объект настроек СКД для сохранения
//
Процедура СохранитьНастройкиУниверсальногоФильтраПортфелейСКД(НастройкиКомпоновки) Экспорт

    Если НастройкиКомпоновки = Неопределено Тогда
        НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено;
        Возврат;
    КонецЕсли;

    НастройкиУниверсальногоФильтраПортфелейСКД = Новый ХранилищеЗначения(НастройкиКомпоновки);

КонецПроцедуры

// Описание: Возвращает сохраненные настройки универсального СКД-фильтра
//
// Возвращаемое значение:
// НастройкиКомпоновкиДанных, Неопределено
//
Функция ПолучитьНастройкиУниверсальногоФильтраПортфелейСКД() Экспорт

    Если НЕ ЗначениеЗаполнено(НастройкиУниверсальногоФильтраПортфелейСКД) Тогда
        Возврат Неопределено;
    КонецЕсли;

    Попытка
        Возврат НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
    Исключение
        ЗаписьЖурналаРегистрации(
            "УниверсальныйФильтрПортфелейСКД",
            УровеньЖурналаРегистрации.Ошибка,
            ,
            ,
            "Не удалось прочитать сохраненные настройки СКД: " + ОписаниеОшибки());
    КонецПопытки;

    Возврат Неопределено;

КонецФункции

// Описание: Очищает настройки универсального фильтра СКД в объекте справочника
//
Процедура СброситьНастройкиУниверсальногоФильтраПортфелейСКД() Экспорт
    НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено;
КонецПроцедуры
```

## 2) `Catalogs/НастройкиЗакрытияПортфелейПоРасписанию/Forms/ФормаЭлемента/Ext/Form/Module.bsl`

Инициализация и запись по образцу формы элемента эталонного справочника в демо конфигурации (ВХ схемы, загрузка из хранилища, копирование).

```bsl
#Область УниверсальныйФильтрПортфелейСКД

&НаСервере
Процедура ИнициализироватьКомпоновщикУниверсальногоФильтраПортфелейСКД()

    ДанныеОбъект = РеквизитФормыВЗначение("Объект");
    ЭтоКопирование = Не Параметры.ЗначениеКопирования.Пустая();

    СхемаКомпоновки = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");
    АдресСхемы = ПоместитьВоВременноеХранилище(СхемаКомпоновки, УникальныйИдентификатор);
    ИсточникНастроек = Новый ИсточникДоступныхНастроекКомпоновкиДанных(АдресСхемы);

    КомпоновщикУниверсальногоФильтраПортфелейСКД.Инициализировать(ИсточникНастроек);

    Если ЭтоКопирование Тогда
        СохраненнаяНастройка = Параметры.ЗначениеКопирования.НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
    Иначе
        СохраненнаяНастройка = ДанныеОбъект.НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
    КонецЕсли;

    Если ЗначениеЗаполнено(СохраненнаяНастройка)
        И ТипЗнч(СохраненнаяНастройка) = Тип("НастройкиКомпоновкиДанных") Тогда
        КомпоновщикУниверсальногоФильтраПортфелейСКД.ЗагрузитьНастройки(СохраненнаяНастройка);
    Иначе
        КомпоновщикУниверсальногоФильтраПортфелейСКД.ЗагрузитьНастройки(СхемаКомпоновки.НастройкиПоУмолчанию);
    КонецЕсли;

КонецПроцедуры

&НаСервере
Процедура ПриСозданииНаСервереПослеУниверсальногоФильтраПортфелей(Отказ, СтандартнаяОбработка)
    ФильтрУниверсальногоОтбораПортфелейСброшен = Ложь;
    ИнициализироватьКомпоновщикУниверсальногоФильтраПортфелейСКД();
КонецПроцедуры

&НаСервере
Процедура ПриЧтенииНаСервереУниверсальныйФильтрПортфелей(ТекущийОбъект)
    ФильтрУниверсальногоОтбораПортфелейСброшен = Ложь;
    ИнициализироватьКомпоновщикУниверсальногоФильтраПортфелейСКД();
КонецПроцедуры

&НаСервере
Процедура ПередЗаписьюНаСервереУниверсальныйФильтрПортфелей(Отказ, ТекущийОбъект, ПараметрыЗаписи)

    Если ФильтрУниверсальногоОтбораПортфелейСброшен Тогда
        ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено;
        ФильтрУниверсальногоОтбораПортфелейСброшен = Ложь;
        Возврат;
    КонецЕсли;

    ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД =
        Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.Настройки);

КонецПроцедуры

&НаКлиенте
Процедура ОчиститьУниверсальныйФильтрПортфелейСКД(Команда)
    ОчиститьУниверсальныйФильтрПортфелейСКДНаСервере();
КонецПроцедуры

&НаСервере
Процедура ОчиститьУниверсальныйФильтрПортфелейСКДНаСервере()
    Объект.НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено;
    ФильтрУниверсальногоОтбораПортфелейСброшен = Истина;
    ИнициализироватьКомпоновщикУниверсальногоФильтраПортфелейСКД();
КонецПроцедуры

#КонецОбласти
```

**Важно:** имена процедур `ПриСозданииНаСервереПосле...` приведены как вставки; в конфигураторе их нужно либо вызвать из существующих `ПриСозданииНаСервере` / `ПриЧтенииНаСервере` / `ПередЗаписьюНаСервере`, либо подписать в форме отдельными обработчиками (в т.ч. через `&После` в расширении).

Если после команды «Очистить» пользователь снова настраивает отбор и сохраняет элемент, нужно сбрасывать `ФильтрУниверсальногоОтбораПортфелейСброшен = Ложь` (например в обработчике изменения таблицы отбора или при первом редактировании компоновщика), иначе в БД запишется `Неопределено` вместо новых настроек.

Имя элемента таблицы `КомпоновщикУниверсальногоФильтраПортфелейСКДНастройкиОтбор` заменить на фактическое имя из `Form.xml`.

## 3) `Catalogs/Портфели/Ext/ManagerModule.bsl`

Одна точка входа: на входе — хранилище настроек, признак ограничения кандидатами, массив ссылок; на выходе — массив ссылок после СКД (или `Неопределено` при ошибке / отсутствии настроек). Дата дня в параметры схемы **не передаётся**.

```bsl
// Описание: Выполняет СКД по макету ДействующиеПортфели с учетом списка кандидатов (дата уже учтена при формировании кандидатов)
//
// Параметры:
// НастройкиУниверсальногоФильтраПортфелейСКД - ХранилищеЗначения, НастройкиКомпоновкиДанных, Неопределено
// ОграничитьКандидатами - Булево - Истина: в выборку попадают только ссылки из КандидатыПортфели
// КандидатыПортфели - Массив из СправочникСсылка.Портфели
//
// Возвращаемое значение:
// Массив - ссылки портфелей после СКД; Неопределено - фильтр не задан или ошибка (см. ЖР)
//
Функция ПортфелиПоНастройкеСКДСКонтекстом(НастройкиУниверсальногоФильтраПортфелейСКД, ОграничитьКандидатами, КандидатыПортфели) Экспорт

    Если НЕ ЗначениеЗаполнено(НастройкиУниверсальногоФильтраПортфелейСКД) Тогда
        Возврат Неопределено;
    КонецЕсли;

    Попытка
        Если ТипЗнч(НастройкиУниверсальногоФильтраПортфелейСКД) = Тип("ХранилищеЗначения") Тогда
            НастройкиПользователя = НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
        Иначе
            НастройкиПользователя = НастройкиУниверсальногоФильтраПортфелейСКД;
        КонецЕсли;
    Исключение
        ЗаписьЖурналаРегистрации("УниверсальныйФильтрПортфелейСКД", УровеньЖурналаРегистрации.Ошибка, , ,
            "Ошибка чтения хранилища СКД: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

    Если НастройкиПользователя = Неопределено Или ТипЗнч(НастройкиПользователя) <> Тип("НастройкиКомпоновкиДанных") Тогда
        Возврат Неопределено;
    КонецЕсли;

    Попытка

        ТаблицаЗначений = Новый ТаблицаЗначений;
        СКД = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");

        КомпоновщикНастроек = Новый КомпоновщикНастроекКомпоновкиДанных;
        КомпоновщикНастроек.Инициализировать(Новый ИсточникДоступныхНастроекКомпоновкиДанных(СКД));
        КомпоновщикНастроек.ЗагрузитьНастройки(СКД.НастройкиПоУмолчанию);
        КомпоновщикНастроек.ЗагрузитьНастройки(НастройкиПользователя);

        НастройкиКомпоновщика = КомпоновщикНастроек.ПолучитьНастройки();

        УстановитьПараметрСКД(НастройкиКомпоновщика, "ОграничитьКандидатами", ОграничитьКандидатами);
        УстановитьПараметрСКД(НастройкиКомпоновщика, "КандидатыПортфели", КандидатыПортфели);

        КомпоновщикМакета = Новый КомпоновщикМакетаКомпоновкиДанных;
        МакетКомпоновкиДанных = КомпоновщикМакета.Выполнить(
            СКД,
            НастройкиКомпоновщика,
            ,
            ,
            Тип("ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений"));

        ПроцессорКомпоновкиДанных = Новый ПроцессорКомпоновкиДанных;
        ПроцессорКомпоновкиДанных.Инициализировать(МакетКомпоновкиДанных);

        ПроцессорВывода = Новый ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений;
        ПроцессорВывода.УстановитьОбъект(ТаблицаЗначений);
        ПроцессорВывода.Вывести(ПроцессорКомпоновкиДанных);

        Возврат ТаблицаЗначений.ВыгрузитьКолонку("Ссылка");

    Исключение
        ЗаписьЖурналаРегистрации("УниверсальныйФильтрПортфелейСКД", УровеньЖурналаРегистрации.Ошибка, , ,
            "Ошибка выполнения СКД портфелей: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

КонецФункции

// Описание: Устанавливает значение параметра данных компоновки по имени
//
// Параметры:
// НастройкиКомпоновщика - НастройкиКомпоновкиДанных
// ИмяПараметра - Строка
// ЗначениеПараметра - Произвольный
//
Процедура УстановитьПараметрСКД(НастройкиКомпоновщика, ИмяПараметра, ЗначениеПараметра)

    ПараметрДанных = НастройкиКомпоновщика.ПараметрыДанных.Элементы.Найти(ИмяПараметра);
    Если ПараметрДанных <> Неопределено Тогда
        ПараметрДанных.Значение = ЗначениеПараметра;
        ПараметрДанных.Использование = Истина;
    КонецЕсли;

КонецПроцедуры
```

Если в конфигурации уже есть `ОбщегоНазначения.УстановитьПараметрыСКД`, можно вызывать её вместо локальной `УстановитьПараметрСКД` по одному параметру за вызов.

**Сценарий «по всем портфелям в рамках макета»** (без ограничения списком кандидатов):

```bsl
Результат = Справочники.Портфели.ПортфелиПоНастройкеСКДСКонтекстом(
    НастройкиХранилище,
    Ложь,
    Новый Массив);
```

## 4) `DataProcessors/ГрупповоеВыполнениеЗакрытияПериодов/Ext/ObjectModule.bsl`

### 4.1. Перенос из настройки

```bsl
    ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД = НастройкаРегулярногоЗакрытия.НастройкиУниверсальногоФильтраПортфелейСКД;
```

### 4.2. Обёртка для обработки

```bsl
// Описание: Применяет сохраненную настройку СКД к списку портфелей-кандидатов (без ручного пересечения; дата не передаётся в СКД)
//
// Параметры:
// МассивПортфелейКандидатов - Массив из СправочникСсылка.Портфели
//
// Возвращаемое значение:
// Массив - отфильтрованные ссылки
//
Функция ПрименитьУниверсальныйФильтрПортфелейСКД(МассивПортфелейКандидатов)

    Если НЕ ЗначениеЗаполнено(ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД) Тогда
        Возврат МассивПортфелейКандидатов;
    КонецЕсли;

    Если НЕ ЗначениеЗаполнено(МассивПортфелейКандидатов) Тогда
        Возврат Новый Массив;
    КонецЕсли;

    РезультатСКД = Справочники.Портфели.ПортфелиПоНастройкеСКДСКонтекстом(
        ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД,
        Истина,
        МассивПортфелейКандидатов);

    Если РезультатСКД = Неопределено Тогда
        Возврат МассивПортфелейКандидатов;
    КонецЕсли;

    Возврат РезультатСКД;

КонецФункции
```

### 4.3. Вызов после базового отбора портфелей

После строки получения действующих портфелей:

```bsl
    ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(ДатаЗакрытия, СписокТипов, СписокПортфелейПоСтратегиям);
    ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

В ветках по границам дата по-прежнему участвует только в `ДействующиеПортфели(День, ...)`, затем:

```bsl
    ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

### 4.4. Форма обработки: синхронизация компоновщика в объект перед исполнением

**Контекст WIM_FIn:** типовая форма `ГрупповоеВыполнениеЗакрытияПериодов` вызывает модуль объекта через **`ТекущийОбъект()`** (объект из `РеквизитФормыВЗначение("Объект")`). Реквизиты обработки с привязкой `DataPath = Объект.…` при серверном вызове уже входят в этот объект — отдельный перенос не нужен. **Компоновщик** — реквизит **только формы**, в состав `Объекта` не входит, поэтому его нужно вручную слить в **`НастройкиУниверсальногоФильтраПортфелейСКД`** того же объекта, который затем обрабатывает `ТекущийОбъект()` (удобно записывать в `ТекущийОбъект().НастройкиУниверсальногоФильтраПортфелейСКД` сразу перед вызовами `ТекущийОбъект().ЗаполнитьПериодыПортфелей()` и т.п.).

Реквизит формы `КомпоновщикУниверсальногоФильтраПортфелейСКД` и реквизит объекта `НастройкиУниверсальногоФильтраПортфелейСКД` **не связаны автоматически**. Пока на сервере не выполнен перенос, `ПрименитьУниверсальныйФильтрПортфелейСКД` в модуле объекта читает **старое** содержимое хранилища (или пусто).

В начале каждой серверной процедуры, которая после этого вызывает код объекта обработки, формирующий списки портфелей (типично — обработчик команды «Выполнить» / «Заполнить» и т.п., по факту разметки формы WIM_FIn), вызывайте перенос:

```bsl
// Описание: Переносит настройки компоновщика с формы обработки в реквизит объекта перед вызовами логики списка портфелей
//
// Параметры: нет (используются реквизиты формы Объект и КомпоновщикУниверсальногоФильтраПортфелейСКД)
//
&НаСервере
Процедура СинхронизироватьУниверсальныйФильтрПортфелейСФормыВОбъект()

    Если КомпоновщикУниверсальногоФильтраПортфелейСКД = Неопределено Тогда
        Объект.НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено;
    Иначе
        Объект.НастройкиУниверсальногоФильтраПортфелейСКД =
            Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.ПолучитьНастройки());
    КонецЕсли;

КонецПроцедуры
```

Инициализацию компоновщика при открытии формы обработки выполняйте так же, как на форме элемента справочника: макет `ДействующиеПортфели`, загрузка из `Объект.НастройкиУниверсальногоФильтраПортфелейСКД`, если хранилище заполнено (после `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия` объект уже может содержать копию из справочника).

Если одна команда на клиенте вызывает несколько серверных процедур подряд, достаточно одной синхронизации **перед первой** из них, которая приводит к расчёту портфелей, либо вынести синхронизацию в общую серверную «обёртку» перед вызовом логики объекта.

## 5) Что проверить после вставки

1. В макете объявлены и используются параметры `ОграничитьКандидатами` и `КандидатыПортфели`; при пути регламента с кандидатами запрос не должен требовать параметр даты (см. `01_detailed_spec.md`, п. 6.1).
2. Компиляция формы: тип реквизита `SettingsComposer`, дерево отбора открывается.
3. Запись элемента справочника: в БД попадает хранилище настроек или `Неопределено` после очистки (флаг `ФильтрУниверсальногоОтбораПортфелейСброшен`).
4. Регламент: количество портфелей после вызова совпадает с ожидаемым результатом СКД по тому же макету.
5. Форма обработки: перед интерактивным запуском цепочки, где вызывается `ПрименитьУниверсальныйФильтрПортфелейСКД`, вызывается синхронизация из п. 4.4; без неё изменения в дереве отбора на форме не влияют на объект.

Старый вариант с функцией `СписокПортфелейПоСохраненнойНастройкеСКД` и ручным пересечением в обработке **заменяется** описанным выше.

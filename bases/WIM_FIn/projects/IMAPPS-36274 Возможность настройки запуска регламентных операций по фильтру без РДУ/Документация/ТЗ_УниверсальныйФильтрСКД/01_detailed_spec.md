# Подробное техническое задание

## 1. Цель доработки

Добавить в справочник `НастройкиЗакрытияПортфелейПоРасписанию` универсальный пользовательский отбор по полям `Справочник.Портфели` на базе СКД по **тому же принципу, что справочник `КонтрольПоручений` в SBB_Depo**:

- в **объекте** хранятся сериализованные настройки СКД в `ХранилищеЗначения` (аналог `НастройкиСКД` у контроля поручений);
- на **форме элемента** располагается реквизит типа `КомпоновщикНастроекКомпоновкиДанных` и таблица дерева отбора `...Settings.Filter` (как `КомпоновщикСхемы` у контроля поручений);
- при выполнении регламента СКД получает **параметры контекста** только для сужения по уже отобранным портфелям (список кандидатов); **отдельный параметр даты дня в схему СКД не передаётся** — дата учтена на этапе вызова `ДействующиеПортфели` и аналогичной логики до применения СКД.

Отличие от старой редакции ТЗ: **не выполнять пересечение массивов в модуле обработки**. Сужение «базовый список портфелей + пользовательский отбор СКД» задаётся **в запросе набора данных макета** через параметры (по аналогии с тем, как в контроле поручений запрос сужается по `&КонтролируемоеПоручение`).

## 3. Новые реквизиты метаданных

### 3.1. В справочнике `НастройкиЗакрытияПортфелейПоРасписанию`

Один реквизит:

- `НастройкиУниверсальногоФильтраПортфелейСКД` (`ХранилищеЗначения`) — сохранённые `НастройкиКомпоновкиДанных`. Пустое или неиспользуемое хранилище означает: дополнительный отбор СКД **не применяется**.

Отдельного булева признака **«использовать фильтр»** и реквизита **строкового представления** фильтра **нет**.

### 3.2. В обработке `ГрупповоеВыполнениеЗакрытияПериодов`

- `НастройкиУниверсальногоФильтраПортфелейСКД` (`ХранилищеЗначения`) — копия из элемента расписания на время выполнения.

## 4. Форма элемента справочника (как у `КонтрольПоручений`)

### 4.1. Реквизиты формы (не путать с реквизитами объекта)

Добавить реквизит формы:

| Имя | Тип в конфигураторе | XML-тип |
|-----|---------------------|---------|
| `КомпоновщикУниверсальногоФильтраПортфелейСКД` | `КомпоновщикНастроекКомпоновкиДанных` | `dcsset:SettingsComposer` |

В форме в шапке `Form` подключить пространства имён `dcsset` (и при необходимости `dcscor`), как в `КонтрольПоручений.ФормаЭлемента`.

### 4.2. Элементы формы

1. **Таблица (дерево)** — привязка `DataPath = КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter`, представление дерева, начальное раскрытие уровней, режим отображения настроек СКД.
2. **Кнопка** «Очистить универсальный фильтр» — команда: обнулить `Объект.НастройкиУниверсальногоФильтраПортфелейСКД`, заново инициализировать компоновщик с настройками по умолчанию макета.

**Не использовать** отдельное модальное окно как основной способ настройки: редактирование отбора — на форме элемента, в дереве отбора.

### 4.3. События формы (XML)

Подписать:

- `OnCreateAtServer` — инициализация компоновщика (новый элемент, копирование, первое открытие).
- `OnReadAtServer` — та же инициализация после чтения объекта из БД (обязательно для расширений и для совпадения данных формы с БД).
- `BeforeWriteAtServer` — перенос из компоновщика в `ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД` (или `Неопределено`, если фильтр очищен по правилам очистки).

## 5. Логика модуля формы (инициализация и запись)

### 5.1. Инициализация при открытии (образец `КонтрольПоручений`)

На сервере:

1. Получить `СхемаКомпоновки = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели")`.
2. Поместить схему во временное хранилище: `Адрес = ПоместитьВоВременноеХранилище(СхемаКомпоновки, УникальныйИдентификатор)`.
3. `ИсточникНастроек = Новый ИсточникДоступныхНастроекКомпоновкиДанных(Адрес)`.
4. `КомпоновщикУниверсальногоФильтраПортфелейСКД.Инициализировать(ИсточникНастроек)`.
5. Если открытие **копирования** — взять сохранённые настройки из `Параметры.ЗначениеКопирования.НастройкиУниверсальногоФильтраПортфелейСКД.Получить()`, иначе из `РеквизитФормыВЗначение("Объект").НастройкиУниверсальногоФильтраПортфелейСКД.Получить()`.
6. Если прочитано значение типа `НастройкиКомпоновкиДанных` — `ЗагрузитьНастройки(Сохраненные)`, иначе — `ЗагрузитьНастройки(СхемаКомпоновки.НастройкиПоУмолчанию)`.

Процедуру инициализации вызывать из `ПриСозданииНаСервере` и из `ПриЧтенииНаСервере`.

### 5.2. Запись

В `ПередЗаписьюНаСервере`:

- при полной очистке фильтра пользователем — `ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД = Неопределено` (или иное согласованное правило «нет фильтра»). Так как после очистки компоновщик снова содержит настройки по умолчанию, без дополнительного признака их можно ошибочно записать в хранилище; в шаблонах в `02_code_templates.md` используется скрытый реквизит формы `ФильтрУниверсальногоОтбораПортфелейСброшен`;
- иначе — `ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД = Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.Настройки)` (или через `ПолучитьНастройки()` по стандартам конфигурации).

## 6. Макет `ДействующиеПортфели` и менеджер `Портфели`

### 6.1. Параметры схемы СКД (обязательная доработка макета)

В запрос набора данных добавить условие с параметрами контекста выполнения из обработки (имена зафиксировать в коде и в схеме). **Параметра даты периода / дня в схеме для регламента нет** — дата уже используется при формировании массива кандидатов до вызова СКД.

| Параметр | Тип | Назначение |
|----------|-----|------------|
| `ОграничитьКандидатами` | Булево | `Ложь` — выборка по всем портфелям, попадающим в базовый запрос макета (если такой режим нужен); `Истина` — только ссылки из `КандидатыПортфели`. |
| `КандидатыПортфели` | массив ссылок на `Справочник.Портфели` | Используется при `ОграничитьКандидатами = Истина`. |

Пример фрагмента условия в запросе (уточнить имя поля ссылки в вашем тексте запроса):

```sql
И (НЕ &ОграничитьКандидатами ИЛИ Портфели.Ссылка В (&КандидатыПортфели))
```

Правила:

- при `ОграничитьКандидатами = Истина` и пустом массиве кандидатов результат компоновки — пустой (ни одного портфеля);
- при `Ложь` параметр `КандидатыПортфели` не используется логикой запроса.

Текст запроса набора при `ОграничитьКандидатами = Истина` должен опираться на переданные ссылки (и пользовательский отбор СКД), **без обязательного условия по дате в параметрах схемы** — актуальность на день уже обеспечена при вызове `ДействующиеПортфели` до СКД.

Так повторяется идея контроля поручений: **сначала сужение контекста параметрами** (для портфелей — список кандидатов), затем пользовательский отбор из сохранённых настроек.

### 6.2. Экспортная функция менеджера `Справочники.Портфели`

Заменить точку расширения на одну функцию (имя согласовать в коде, например):

`ПортфелиПоНастройкеСКДСКонтекстом(НастройкиУниверсальногоФильтраПортфелейСКД, ОграничитьКандидатами, КандидатыПортфели)`

**Алгоритм:**

1. Если `НастройкиУниверсальногоФильтраПортфелейСКД` пусто или не читается — вернуть `Неопределено` (вызывающий код не меняет список).
2. Загрузить макет `ДействующиеПортфели`.
3. Инициализировать компоновщик (как в текущих шаблонах), загрузить `НастройкиПоУмолчанию`, затем сохранённые пользовательские настройки.
4. Получить `НастройкиКомпоновщика = КомпоновщикНастроек.ПолучитьНастройки()`.
5. Установить параметры данных:
   - `ОграничитьКандидатами` = переданный флаг;
   - `КандидатыПортфели` = переданный массив (если флаг `Ложь`, значение может быть пустым массивом — запрос не должен от него зависеть).
6. Выполнить компоновку в коллекцию значений, вернуть массив из колонки `Ссылка` (или фактическое имя поля ссылки в макете).

Ошибки выполнения — в ЖР; при ошибке вернуть `Неопределено`, чтобы обработка применила политику fallback (как в тест-кейсах).

## 7. Обработка `ГрупповоеВыполнениеЗакрытияПериодов`

### 7.1. Перенос из настроек

При заполнении из справочника переносить `НастройкиУниверсальногоФильтраПортфелейСКД`.

### 7.2. Одна функция-обёртка

Ввести функцию модуля объекта обработки, например:

`ПрименитьУниверсальныйФильтрПортфелейСКД(МассивПортфелейКандидатов)`

**Алгоритм:**

1. Если хранилище `НастройкиУниверсальногоФильтраПортфелейСКД` пустое — вернуть `МассивПортфелейКандидатов` без изменений.
2. Вызвать  
   `ПортфелиПоНастройкеСКДСКонтекстом(ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД, Истина, МассивПортфелейКандидатов)`.
3. Если результат `Неопределено` (ошибка или нет данных) — вернуть исходный массив кандидатов (fallback, см. тесты).
4. Иначе вернуть полученный массив (уже с учётом СКД и ограничения кандидатами).

Для сценария **«по всем портфелям в рамках макета»** (без предварительного списка от типов/стратегий) вызывать ту же функцию менеджера с `ОграничитьКандидатами = Ложь` и пустым массивом кандидатов — если такой сценарий появится в коде закрытия (без параметра даты в СКД).

### 7.3. Места вызова

1. В `ЗаполнитьСписокПортфелей` после получения массива `ДействующиеПортфели` от типовой логики:  
   `ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);`
2. В ветках `ЗаполнитьПериодыПортфелейПоГраницам` и `ЗаполнитьПериодыПортфелейПоВарианту_ДНГ_ДКР` — после формирования списка портфелей на дату:  
   `ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);`

Пересечение массивов вручную (`Соответствие` + цикл) **не использовать** — его заменяет параметр `ОграничитьКандидатами` и условие в запросе СКД.

### 7.4. Форма обработки и передача фильтра в исполнение

На форме обработки `ГрупповоеВыполнениеЗакрытияПериодов` добавить реквизит `КомпоновщикУниверсальногоФильтраПортфелейСКД` (тип компоновщика), таблицу отбора `...Settings.Filter`, инициализацию по макету `ДействующиеПортфели` при открытии из `Объект.НастройкиУниверсальногоФильтраПортфелейСКД` (после переноса из справочника или пусто).

**Как соединяются изменения на форме с модулем объекта (WIM_FIn).** Обычные реквизиты обработки с `DataPath = Объект.…` при серверном вызове уже попадают в объект, который форма передаёт в `РеквизитФормыВЗначение("Объект")` и в типовой функции **`ТекущийОбъект()`**; отдельно их «в исполнение» не пробрасывают. Редактирование дерева отбора меняет только реквизит **формы** `КомпоновщикУниверсальногоФильтраПортфелейСКД`. Функция `ПрименитьУниверсальныйФильтрПортфелейСКД` в модуле **объекта** читает `ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД`. Поэтому перед `ТекущийОбъект().ЗаполнитьПериодыПортфелей()` (и любыми вызовами, где внутри объекта уже идёт отбор портфелей) выполняют синхронизацию: поместить `КомпоновщикУниверсальногоФильтраПортфелейСКД.ПолучитьНастройки()` (или `.Настройки`) в хранилище объекта, в т.ч. в **`ТекущийОбъект().НастройкиУниверсальногоФильтраПортфелейСКД`**; при сбросе — `Неопределено`. Пример процедуры — `02_code_templates.md`, п. 4.4.

**Регламент:** объект обработки создаётся без открытия формы; хранилище попадает в объект из `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия` — отдельная синхронизация с формы не нужна.

## 8. Пошаговый порядок реализации

1. Доработать макет `ДействующиеПортфели` (параметры и условие в запросе).
2. Добавить в менеджер `Портфели` функцию `ПортфелиПоНастройкеСКДСКонтекстом`.
3. Добавить реквизиты справочника и обработки (если ещё не добавлены).
4. Доработать `Form.xml` элемента настроек: реквизит компоновщика, таблица `Filter`, события.
5. Реализовать модуль формы: инициализация, чтение, запись, очистка.
6. При необходимости — модуль объекта справочника (только вспомогательные процедуры чтения/записи хранилища, без реквизита представления).
7. Внедрить в обработку обёртку и вызовы в местах заполнения портфелей.
8. Доработать форму обработки: компоновщик, синхронизация в объект перед исполнением (см. п. 7.4).
9. Тесты.

## 9. Тест-кейсы

1. Сохранение и повторное открытие элемента: дерево отбора совпадает с сохранённым.
2. Базовый отбор даёт 100 портфелей, СКД оставляет 30 при тех же кандидатах — в `ПериодыПортфелей` не более 30.
3. Пустое хранилище настроек СКД — поведение как до доработки.
4. Битые настройки СКД — запись в ЖР, fallback без изменения списка (или по согласованию).
5. `ОграничитьКандидатами = Истина`, пустой массив кандидатов — пустой результат.
6. Многопоточный и интерактивный режимы.
7. Интерактивно: изменение отбора на форме обработки и выполнение команды после синхронизации даёт тот же результат СКД, что и при переносе того же хранилища из справочника без ручного редактирования на форме.

## 10. Критерии приемки

1. Настройка отбора на форме элемента справочника через встроенный компоновщик.
2. Настройка и изменение отбора на форме обработки; перед исполнением фильтр передаётся в объект обработки (синхронизация).
3. Регламент получает итоговый список портфелей через одну функцию менеджера; в схему СКД передаются параметры ограничения кандидатами, **без параметра даты дня**.
4. При незаполненном хранилище настроек СКД поведение не меняется относительно версии до доработки.
5. Нет обязательного ручного пересечения массивов в коде обработки.

# Подробное техническое задание

## 1. Цель доработки

Добавить в справочник `НастройкиЗакрытияПортфелейПоРасписанию` универсальный пользовательский отбор по полям `Справочник.Портфели` на базе СКД:
- пользователь на форме настройки открывает компоновщик СКД;
- задает произвольные фильтры по полям Портфеля;
- видит на форме краткое представление активных фильтров;
- сохраняет элемент справочника;
- при регламентном запуске обработки `ГрупповоеВыполнениеЗакрытияПериодов` эти фильтры применяются как дополнительное пересечение с уже отобранными портфелями.

## 3. Новые реквизиты метаданных

### 3.1. В справочнике `НастройкиЗакрытияПортфелейПоРасписанию`

Добавить реквизиты:

1. `ИспользоватьУниверсальныйФильтрСКД` (Булево, по умолчанию `Ложь`)
2. `НастройкиУниверсальногоФильтраСКД` (ХранилищеЗначения)
3. `ПредставлениеУниверсальногоФильтраСКД` (Строка, неограниченная длина)

Назначение:
- `Использовать...` — feature-toggle.
- `Настройки...` — сериализованные настройки компоновщика.
- `Представление...` — текст для формы (что именно отфильтровано).

### 3.2. В обработке `ГрупповоеВыполнениеЗакрытияПериодов`

Добавить реквизиты:

1. `ИспользоватьУниверсальныйФильтрСКД` (Булево)
2. `НастройкиУниверсальногоФильтраСКД` (ХранилищеЗначения)

Назначение: перенос сохраненных настроек из справочника в объект обработки для регламентного расчета.

## 4. Изменения формы элемента справочника

## 4.1. Элементы формы

В `ФормаЭлемента` добавить:

1. Флаг: `ИспользоватьУниверсальныйФильтрСКД`
2. Команда/кнопка: `НастроитьУниверсальныйФильтрСКД`
3. Команда/кнопка: `ОчиститьУниверсальныйФильтрСКД`
4. Поле текста (только просмотр): `ПредставлениеУниверсальногоФильтраСКД`

Расположение: в блоке фильтров рядом с `ТипыКлиентов`, `Стратегии`.

## 4.2. Поведение формы

1. При открытии:
   - если фильтр включен и настройки не пустые, отобразить `ПредставлениеУниверсальногоФильтраСКД`.
2. При нажатии `Настроить...`:
   - открыть форму настройки СКД на базе макета `Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели")`;
   - загрузить текущие сохраненные настройки, если есть;
   - после ОК сохранить настройки в `НастройкиУниверсальногоФильтраСКД`;
   - пересчитать `ПредставлениеУниверсальногоФильтраСКД`.
3. При нажатии `Очистить...`:
   - очистить хранилище и представление.
4. В `УправлениеФормой()`:
   - доступность кнопок и представления зависит от флага `ИспользоватьУниверсальныйФильтрСКД`.

## 5. Изменения модуля объекта справочника

В `ObjectModule.bsl` добавить сервисные процедуры/функции:

1. `СохранитьНастройкиУниверсальногоФильтраСКД(НастройкиКомпоновки)`
2. `ПолучитьНастройкиУниверсальногоФильтраСКД()`
3. `СброситьНастройкиУниверсальногоФильтраСКД()`
4. `СформироватьПредставлениеУниверсальногоФильтраСКД(НастройкиКомпоновки)`

Логика:
- хранение через `Новый ХранилищеЗначения(...)`;
- извлечение через `Получить()`;
- представление строится из элементов `Отбор.Элементы` с учетом `Использование = Истина`.

## 6. Изменения менеджера `Справочники.Портфели`

В `ManagerModule.bsl` добавить экспортную функцию:

- `СписокПортфелейПоСохраненнойНастройкеСКД(ДатаПериода, НастройкиУниверсальногоФильтраСКД)`

Алгоритм:

1. Загрузить макет `ДействующиеПортфели`.
2. Инициализировать компоновщик.
3. Загрузить `НастройкиПоУмолчанию`.
4. Подмешать сохраненные пользовательские настройки.
5. Установить параметр `ДатаПериода`.
6. Выполнить СКД в коллекцию значений.
7. Вернуть массив `Ссылка`.

Важно:
- если настройки пусты/не читаются — вернуть `Неопределено` (чтобы не ломать базовую логику);
- все ошибки логировать в ЖР.

## 7. Изменения обработки `ГрупповоеВыполнениеЗакрытияПериодов`

## 7.1. Перенос реквизитов из справочника

В процедуре `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия` после текущих присваиваний добавить перенос:

- `ЭтотОбъект.ИспользоватьУниверсальныйФильтрСКД = НастройкаРегулярногоЗакрытия.ИспользоватьУниверсальныйФильтрСКД;`
- `ЭтотОбъект.НастройкиУниверсальногоФильтраСКД = НастройкаРегулярногоЗакрытия.НастройкиУниверсальногоФильтраСКД;`

## 7.2. Новая функция дополнительной фильтрации

Добавить в модуль объекта функцию:

- `ОтфильтроватьМассивПортфелейУниверсальнымФильтромСКД(ДатаЗакрытия, МассивПортфелейИсходный)`

Алгоритм:

1. Если фильтр выключен/пустой — вернуть исходный массив.
2. Получить массив из СКД:
   - `Справочники.Портфели.СписокПортфелейПоСохраненнойНастройкеСКД(ДатаЗакрытия, ЭтотОбъект.НастройкиУниверсальногоФильтраСКД)`.
3. Если массив СКД пустой — вернуть пустой массив.
4. Вернуть пересечение:
   - `Исходный ∩ СКД`.

## 7.3. Где вызывать дополнительную фильтрацию

1. В `ЗаполнитьСписокПортфелей`:
   - сразу после получения `ДействующиеПортфели`.
2. В ветках по границам:
   - в `ЗаполнитьПериодыПортфелейПоГраницам`;
   - в `ЗаполнитьПериодыПортфелейПоВарианту_ДНГ_ДКР`;
   - применять пересечение перед добавлением строк в `ПериодыПортфелей`.

Правило: универсальный фильтр всегда является дополнительным (`И`) к уже рассчитанным отборам.

## 8. Пошаговый порядок реализации

1. Добавить реквизиты в `Catalogs/НастройкиЗакрытияПортфелейПоРасписанию`.
2. Добавить реквизиты в `DataProcessors/ГрупповоеВыполнениеЗакрытияПериодов`.
3. Доработать форму элемента справочника (элементы, команды, обработчики).
4. Доработать `ObjectModule` справочника (сохранение/чтение/представление).
5. Доработать `ManagerModule` справочника `Портфели` (новая экспортная функция СКД по сохраненной настройке).
6. Доработать `ObjectModule` обработки:
   - перенос реквизитов;
   - функция пересечения;
   - внедрение вызовов в `ЗаполнитьСписокПортфелей` и ветки по границам.
7. Обновить форму обработки (при необходимости показать пользователю текст активного универсального фильтра в интерактивном режиме).
8. Провести тесты.

## 9. Тест-кейсы (обязательно)

1. Сохранение/повторное открытие элемента:
   - фильтр сохранился;
   - представление фильтра отображается.
2. Базовый отбор дает 100 портфелей, СКД-фильтр 30:
   - итог в `ПериодыПортфелей` только 30.
3. СКД-фильтр отключен:
   - поведение полностью старое.
4. СКД-фильтр некорректен/битый:
   - ошибка в ЖР;
   - регламент работает без падения (fallback: без дополнительного фильтра).
5. Проверка многопоточного и обычного режима.

## 10. Критерии приемки

1. Пользователь может на форме настроить фильтр по любому доступному полю Портфеля через СКД.
2. Пользователь видит список условий фильтра в элементе справочника.
3. Регламент учитывает фильтр как дополнительный к существующим отборам.
4. При выключенном флаге новый функционал не влияет на текущую логику.
5. Не возникает ошибок при выполнении регламентного задания.

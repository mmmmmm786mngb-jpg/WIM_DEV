## 0. Методика именования (как в конфигурации заказчика / WIM_FIn)

### 0.3. Параметры данных макета СКД `ДействующиеПортфели`

Имена параметров в схеме и в коде должны совпадать (регистр как в конфигураторе):

| Параметр | Тип | Назначение |
|----------|-----|------------|
| `ОграничитьКандидатами` | Булево | `Истина` — в наборе только ссылки из `КандидатыПортфели` |
| `КандидатыПортфели` | Массив ссылок на `Справочник.Портфели` | Используется при `ОграничитьКандидатами = Истина` |

## 1. Контекст

В информационной базе регламентное групповое закрытие периода по портфелям выполняется обработкой **«Групповое выполнение закрытия периодов»** (`ГрупповоеВыполнениеЗакрытияПериодов`) на основании элементов справочника **«Настройки закрытия портфелей по расписанию»** (`НастройкиЗакрытияПортфелейПоРасписанию`). Отбор портфелей задаётся в том числе по типам клиентов договоров ДУ и по стратегиям. Для договоров РДУ тип клиента часто совпадает с другими договорами (например, физические лица), из‑за чего отбор только по типу не позволяет исключить РДУ без перечисления конкретных договоров или портфелей, что неудобно и трудоёмко.

Требуется дать пользователю **универсальный отбор по реквизитам справочника «Портфели»** на базе **схемы компоновки данных (СКД)** с сохранением настройки в элементе справочника расписания и применением этой настройки при фактическом выполнении закрытия.

---

## 2. Цели доработки

1. Сократить время подготовки и сопровождения регламентных настроек закрытия при большом количестве договоров РДУ.
2. Обеспечить **гибкий пользовательский фильтр** по доступным в СКД полям портфеля без ручного перечисления большого числа элементов.
3. Сохранить **совместимость** с текущим поведением: если дополнительный фильтр СКД **не задан** (хранилище пустое или не используется), логика отбора портфелей не меняется.

---

## 3. Задачи доработки

1. Расширить справочник `НастройкиЗакрытияПортфелейПоРасписанию` **одним** реквизитом `НастройкиУниверсальногоФильтраПортфелейСКД` типа **ХранилищеЗначения** для сериализованных **НастройкиКомпоновкиДанных**. Отдельного булева признака «использовать фильтр» не вводить: факт настройки определяется наличием и содержимым хранилища (и согласованными с заказчиком правилами «пустого» отбора).
2. Доработать **форму элемента** справочника: отбор СКД через реквизит формы `КомпоновщикУниверсальногоФильтраПортфелейСКД` и дерево отбора (`КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter`). Отдельное поле только просмотра для текстового представления фильтра **не предусматривается**.
3. Расширить обработку `ГрупповоеВыполнениеЗакрытияПериодов` реквизитом `НастройкиУниверсальногоФильтраПортфелейСКД` для переноса настроек из элемента расписания и применения фильтра при формировании списков портфелей.
3а. Доработать **форму** обработки `ГрупповоеВыполнениеЗакрытияПериодов`: реквизит формы `КомпоновщикУниверсальногоФильтраПортфелейСКД`, таблица отбора `КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter`, инициализация по макету `ДействующиеПортфели` при открытии, команда очистки по аналогии со справочником. Без синхронизации компоновщика в реквизит **объекта** обработки изменения на форме не попадут в `ПрименитьУниверсальныйФильтрПортфелейСКД` (см. п. 6.0–6.1 и `02_code_templates.md`, п. 4.4).
4. Доработать макет СКД `ДействующиеПортфели` (справочник `Портфели`): параметры `ОграничитьКандидатами`, `КандидатыПортфели` и условие в запросе набора данных; **без** параметра даты дня для передачи из регламента.
5. Реализовать в менеджере справочника `Портфели` экспортную функцию `ПортфелиПоНастройкеСКДСКонтекстом`.
6. Во всех местах модуля объекта обработки, где после штатного отбора формируется массив действующих портфелей, вызывать `ПрименитьУниверсальныйФильтрПортфелейСКД` при непустом хранилище.

## 4. Примеры кода (ориентир для реализации)

Фрагменты приведены в стиле конфигурации заказчика; имена — из п. 0.

### 4.1. Перенос хранилища в обработку

В процедуре `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия` после переноса связанных реквизитов (например после строки с `СоздаватьОбщиеОперации`):

```bsl
    ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД =
        НастройкаРегулярногоЗакрытия.НастройкиУниверсальногоФильтраПортфелейСКД;
```

При обращении к реквизитам через ссылку без чтения объекта допустимо `ОбщегоНазначения.ЗначенияРеквизитовОбъекта` по правилам конфигурации.

### 4.2. Обёртка в модуле объекта обработки

```bsl
// Описание: Применяет сохраненную настройку СКД к списку портфелей-кандидатов
//
// Параметры:
// МассивПортфелейКандидатов - Массив из СправочникСсылка.Портфели
//
// Возвращаемое значение:
// Массив - отфильтрованные ссылки
//
Функция ПрименитьУниверсальныйФильтрПортфелейСКД(МассивПортфелейКандидатов)

    Если НЕ ЗначениеЗаполнено(ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД) Тогда
        Возврат МассивПортфелейКандидатов;
    КонецЕсли;

    Если НЕ ЗначениеЗаполнено(МассивПортфелейКандидатов) Тогда
        Возврат Новый Массив;
    КонецЕсли;

    РезультатСКД = Справочники.Портфели.ПортфелиПоНастройкеСКДСКонтекстом(
        ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД,
        Истина,
        МассивПортфелейКандидатов);

    Если РезультатСКД = Неопределено Тогда
        Возврат МассивПортфелейКандидатов;
    КонецЕсли;

    Возврат РезультатСКД;

КонецФункции
```

### 4.3. Вызов после `ДействующиеПортфели`

В `ЗаполнитьСписокПортфелей`:

```bsl
    ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(ДатаЗакрытия, СписокТипов, СписокПортфелейПоСтратегиям);
    ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

В ветках с переменной `День` (по границам, варианты периода) — после присваивания `ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(День, ...)`:

```bsl
    ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

### 4.4. Менеджер `Портфели`: выполнение СКД (скелет)

```bsl
// Описание: Выполняет СКД по макету ДействующиеПортфели с ограничением по кандидатам
//
// Параметры:
// НастройкиУниверсальногоФильтраПортфелейСКД - ХранилищеЗначения, Неопределено
// ОграничитьКандидатами - Булево
// КандидатыПортфели - Массив из СправочникСсылка.Портфели
//
// Возвращаемое значение:
// Массив, Неопределено
//
Функция ПортфелиПоНастройкеСКДСКонтекстом(НастройкиУниверсальногоФильтраПортфелейСКД, ОграничитьКандидатами, КандидатыПортфели) Экспорт

    Если НЕ ЗначениеЗаполнено(НастройкиУниверсальногоФильтраПортфелейСКД) Тогда
        Возврат Неопределено;
    КонецЕсли;

    Попытка
        НастройкиПользователя = НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
    Исключение
        ЗаписьЖурналаРегистрации("УниверсальныйФильтрПортфелейСКД", УровеньЖурналаРегистрации.Ошибка, , ,
            "Ошибка чтения хранилища СКД: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

    Если НастройкиПользователя = Неопределено
        Или ТипЗнч(НастройкиПользователя) <> Тип("НастройкиКомпоновкиДанных") Тогда
        Возврат Неопределено;
    КонецЕсли;

    Попытка
        ТаблицаЗначений = Новый ТаблицаЗначений;
        СКД = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");
        КомпоновщикНастроек = Новый КомпоновщикНастроекКомпоновкиДанных;
        КомпоновщикНастроек.Инициализировать(Новый ИсточникДоступныхНастроекКомпоновкиДанных(СКД));
        КомпоновщикНастроек.ЗагрузитьНастройки(СКД.НастройкиПоУмолчанию);
        КомпоновщикНастроек.ЗагрузитьНастройки(НастройкиПользователя);

        НастройкиКомпоновщика = КомпоновщикНастроек.ПолучитьНастройки();
        УстановитьПараметрСКД(НастройкиКомпоновщика, "ОграничитьКандидатами", ОграничитьКандидатами);
        УстановитьПараметрСКД(НастройкиКомпоновщика, "КандидатыПортфели", КандидатыПортфели);

        // КомпоновщикМакета, ПроцессорКомпоновкиДанных, вывод в коллекцию — как в типовых шаблонах заказчика
        Возврат ТаблицаЗначений.ВыгрузитьКолонку("Ссылка");
    Исключение
        ЗаписьЖурналаРегистрации("УниверсальныйФильтрПортфелейСКД", УровеньЖурналаРегистрации.Ошибка, , ,
            "Ошибка выполнения СКД портфелей: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

КонецФункции
```

Полный текст вспомогательной `УстановитьПараметрСКД` и разбор хранилища — по внутреннему ТЗ заказчика (`02_code_templates.md`).

### 4.5. Форма элемента: запись и очистка (скелет)

```bsl
&НаСервере
Процедура ПередЗаписьюНаСервереУниверсальныйФильтрПортфелей(Отказ, ТекущийОбъект, ПараметрыЗаписи)

    Если ФильтрУниверсальногоОтбораПортфелейСброшен Тогда

    ТекущийОбъект.НастройкиУниверсальногоФильтраПортфелейСКД =
        Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.Настройки);

&НаСервере
```

## 5. Объекты метаданных и изменения (сводка)

### 5.1. Справочник `НастройкиЗакрытияПортфелейПоРасписанию`

- Реквизит `НастройкиУниверсальногоФильтраПортфелейСКД` (ХранилищеЗначения).
- Форма элемента: `КомпоновщикУниверсальногоФильтраПортфелейСКД`, дерево отбора, команда очистки, скрытый `ФильтрУниверсальногоОтбораПортфелейСброшен`.
- Модуль объекта справочника: по необходимости вспомогательные методы чтения/записи хранилища **без** отдельного строкового реквизита представления.

### 5.2. Обработка `ГрупповоеВыполнениеЗакрытияПериодов`

- Реквизит объекта `НастройкиУниверсальногоФильтраПортфелейСКД`.
- Форма обработки: `КомпоновщикУниверсальногоФильтраПортфелейСКД`, события и синхронизация в объект перед серверными шагами, которые строят списки портфелей (см. `02_code_templates.md`, раздел 4.4).
- Дополнение `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия`, функция `ПрименитьУниверсальныйФильтрПортфелейСКД`, вызовы после `ДействующиеПортфели` во всех нужных процедурах.

### 5.3. Справочник `Портфели`

- Менеджер: `ПортфелиПоНастройкеСКДСКонтекстом`.

## 6. Принципы реализации (технические)

### 6.0. Как в WIM_FIn настройки с формы обработки попадают в исполнение (общая методика)

В конфигурации **WIM_FIn** у формы обработки `ГрупповоеВыполнениеЗакрытияПериодов` основной контейнер данных — **объект обработки** (`Свойства формы` → основной реквизит `Объект`).

1. **Реквизиты, объявленные у обработки в метаданных** (даты периода, `ПоВсемДействующимПортфелям`, табличные части `ТипыКлиентов`, `Стратегии`, списки значений и т.д.), при привязке полей формы к `Объект.ИмяРеквизита` участвуют в стандартном обмене форма ↔ сервер: при вызове `&НаСервере` платформа передаёт на сервер актуальное состояние `Объект` с клиента. В модуле формы исполнение строится на **`РеквизитФормыВЗначение("Объект")`** и функции **`ТекущийОбъект()`** (в типовом коде — с кэшем `мТекущийОбъектНаСервере`): по полученному объекту вызываются процедуры **модуля объекта** обработки, например `ЗаполнитьПериодыПортфелей()`, `ВыполнитьРегламентныеОперации(...)`. После изменений объекта на сервере вызывают **`ЗначениеВРеквизитФормы(ТекущийОбъект(), "Объект")`**, чтобы вернуть обновлённые данные на форму.

2. **Итог:** «другие реквизиты», которые пользователь меняет на форме и которые **являются реквизитами объекта обработки** с `DataPath` на `Объект`, **уже находятся** в том же объекте, который получает модуль объекта через цепочку формы. Отдельно «передавать» их вручную не нужно — нужно только вызывать методы объекта после того, как с клиента ушёл серверный вызов (что и делает типовая форма).

3. **Реквизиты только формы** (нет соответствующего реквизита в метаданных обработки или не включены в состав передаваемого объекта) в `РеквизитФормыВЗначение("Объект")` **не попадают**. Их значения перед вызовом методов модуля объекта нужно **явно** записать в поля объекта обработки на сервере (в тот же объект, что вернёт `ТекущийОбъект()`, либо в `Объект` до вызова, согласованно с кэшем формы).

4. **Реквизит `НастройкиУниверсальногоФильтраПортфелейСКД`** после добавления в метаданные обработки — такой же **реквизит объекта**, как период или признаки; он будет уезжать на сервер в составе `Объект` **если** в него записано значение (например скопировано из справочника при `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия`). **Дерево отбора СКД** на форме хранится в **реквизите формы** `КомпоновщикУниверсальногоФильтраПортфелейСКД` и **автоматически не сериализуется** в `Объект.НастройкиУниверсальногоФильтраПортфелейСКД`. По той же методике, что и для любого «чисто форменного» состояния, перед `ТекущийОбъект().ЗаполнитьПериодыПортфелей()` (и любыми другими вызовами, внутри которых читается хранилище СКД) выполняют **явный перенос** компоновщика в хранилище объекта (см. п. 6.1 и `02_code_templates.md`, п. 4.4). При необходимости перенос делают в **`ТекущийОбъект().НастройкиУниверсальногоФильтраПортфелейСКД`**, чтобы совпадать с используемым в форме экземпляром объекта.

### 6.1. Синхронизация компоновщика и хранилище (уточнение к п. 6.0)

1. **Хранение:** один реквизит ХранилищеЗначения в объекте справочника и в объекте обработки; на форме элемента справочника и на **форме обработки** — реквизит `КомпоновщикУниверсальногоФильтраПортфелейСКД` типа КомпоновщикНастроекКомпоновкиДанных.
2. **Перенос в объект перед вызовами модуля объекта:** выполнить присвоение в `НастройкиУниверсальногоФильтраПортфелейСКД` из компоновщика (`Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.ПолучитьНастройки())` или свойство `.Настройки` — по стандарту проекта); при очистке — `Неопределено`. Точку размещают в начале серверных процедур формы перед `ТекущийОбъект().ЗаполнитьПериодыПортфелей()` / выполнением закрытия / иными вызовами, где внутри объекта используется фильтр.
3. **Регламент (фон):** форма не открывается; объект обработки заполняется программно, в том числе `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия` переносит хранилище из справочника — дополнительная синхронизация с формы не требуется.

### 6.2. Прочие технические принципы

1. **Инициализация компоновщика:** при создании на сервере и при чтении объекта (справочник и форма обработки) — единая процедура по макету `ДействующиеПортфели`.
2. **Запись элемента справочника:** см. п. 4.5; при очистке — `Неопределено` в реквизите объекта, чтобы регламент не вызывал СКД.
3. **Копирование элемента справочника:** загрузка настроек из `Параметры.ЗначениеКопирования`.
4. **Сужение списка портфелей:** через `ПортфелиПоНастройкеСКДСКонтекстом` без даты в параметрах СКД; ручное пересечение массивов в обработке не дублировать.
5. **Права, производительность, версия платформы** — по общим правилам конфигурации заказчика.

## 7. Критерии приемки

1. В элементе справочника пользователь настраивает отбор в дереве компоновщика, сохраняет элемент и при повторном открытии видит те же условия.
2. На форме обработки пользователь видит и может менять отбор; после синхронизации в объект (перед выполнением) итоговый список портфелей учитывает актуальные настройки СКД.
3. При регламентном запуске в обработку переносится `НастройкиУниверсальногоФильтраПортфелейСКД`; итоговый состав портфелей соответствует штатным отборам плюс СКД, если хранилище заполнено.
4. При незаполненном хранилище поведение обработки совпадает с версией до доработки.
5. Ошибка в настройках СКД или при компоновке не приводит к аварийному завершению сеанса регламента; фиксируется запись в журнале регистрации.

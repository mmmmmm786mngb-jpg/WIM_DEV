IMAPPS-36274: фильтр портфелей по СКД

Краткая схема для разработчика: реквизиты, формы, макет, перенос в обработку, сужение списка портфелей.

2. Макет схемы СКД

  • Схема компоновки данных хранится как макет в справочнике Портфели (в реализации проекта: ДействующиеПортфели).
  • Текст запроса и поля набора должны позволять получить список портфелей, подходящих под отбор (как минимум колонка со ссылкой на портфель, например Ссылка).
  • В схеме для регламента задаются параметры ограничения списка кандидатов; дату дня в параметры СКД не передают — она уже учтена при вызове ДействующиеПортфели до применения фильтра.

3. Форма элемента справочника настроек

Обязательно для штатной работы дерева отбора. В метаданных формы у реквизита КомпоновщикУниверсальногоФильтраПортфелейСКД тип должен быть КомпоновщикНастроекКомпоновкиДанных (в XML-выгрузке формы: dcsset:SettingsComposer). Тип НастройкиКомпоновкиДанных у реквизита формы для привязки таблицы ...Settings.Filter не подходит: дерево отбора на управляемой форме работает корректно только с компоновщиком.

Реквизит формы КомпоновщикУниверсальногоФильтраПортфелейСКД

  • Смысл: держать на форме редактируемое состояние фильтра СКД на время работы формы (объект компоновщика в памяти).
  • В базе данных сохраняется не реквизит формы, а Объект.НастройкиУниверсальногоФильтраПортфелейСКД (хранилище с объектом НастройкиКомпоновкиДанных), получаемым через КомпоновщикУниверсальногоФильтраПортфелейСКД.ПолучитьНастройки().

Элемент формы

  • Таблица (дерево) с DataPath = КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter, режим отображения отбора СКД (как в типовых формах с компоновщиком).
  • При открытии формы: инициализировать компоновщик по макету Портфели и при наличии загрузить сохраненные настройки из Объект.НастройкиУниверсальногоФильтраПортфелейСКД.Получить().
  • Перед записью элемента справочника: Объект.НастройкиУниверсальногоФильтраПортфелейСКД = Новый ХранилищеЗначения(КомпоновщикУниверсальногоФильтраПортфелейСКД.ПолучитьНастройки()) (или очистка, если фильтр сброшен).
  • Для уже записанных элементов после чтения из БД повторно вызывать ту же загрузку в компоновщик (событие чтения на сервере), чтобы данные из хранилища совпали с открытой формой.

Form.xml: подписка на серверные события (имена обработчиков замените на свои или используйте префикс расширения)

<Events>
    <Event name="OnReadAtServer">ИмяМодуля_ПриЧтенииНаСервере</Event>
    <Event name="BeforeWriteAtServer">ИмяМодуля_ПередЗаписьюНаСервере</Event>
</Events>

Дополнительно: подписка &После("ПриСозданииНаСервере") для нового элемента. Для записанных элементов нужна ПриЧтенииНаСервере с той же инициализацией (см. событие в Form.xml).

Module.bsl формы: инициализация компоновщика при открытии (вызывать из ПриСозданииНаСервере и ПриЧтенииНаСервере)

// Описание: Заполняет реквизит формы КомпоновщикУниверсальногоФильтраПортфелейСКД (КомпоновщикНастроекКомпоновкиДанных) из макета и Объект.НастройкиУниверсальногоФильтраПортфелейСКД.
//
&НаСервере
Процедура ИнициализироватьКомпоновщикУниверсальногоФильтраПортфелейСКД()

    ИмяМакетаСКД = "ДействующиеПортфели";
    СхемаКомпоновкиДанных = Справочники.Портфели.ПолучитьМакет(ИмяМакетаСКД);
    КомпоновщикНастроек = Новый КомпоновщикНастроекКомпоновкиДанных;
    КомпоновщикНастроек.Инициализировать(Новый ИсточникДоступныхНастроекКомпоновкиДанных(СхемаКомпоновкиДанных));
    КомпоновщикНастроек.ЗагрузитьНастройки(СхемаКомпоновкиДанных.НастройкиПоУмолчанию);

    Если ТипЗнч(Объект.НастройкиУниверсальногоФильтраПортфелейСКД) = Тип("ХранилищеЗначения") Тогда
        СохраненныеНастройки = Объект.НастройкиУниверсальногоФильтраПортфелейСКД.Получить();
        Если ТипЗнч(СохраненныеНастройки) = Тип("НастройкиКомпоновкиДанных") Тогда
            КомпоновщикНастроек.ЗагрузитьНастройки(СохраненныеНастройки);
        КонецЕсли;
    КонецЕсли;

    КомпоновщикУниверсальногоФильтраПортфелейСКД = КомпоновщикНастроек;

4. Обработка: перенос фильтра из настроек

После выбора настройки регламентного закрытия (когда заполняются реквизиты обработки по настройке) скопировать:
Обработка.НастройкиУниверсальногоФильтраПортфелейСКД = Настройка.НастройкиУниверсальногоФильтраПортфелейСКД

Так пользователь видит на обработке тот же фильтр, что сохранен в элементе справочника настроек, без отдельного ввода.

5. Форма обработки (по необходимости)

Если требуется показывать или менять отбор на форме обработки, применяют ту же схему: реквизит КомпоновщикУниверсальногоФильтраПортфелейСКД (компоновщик), таблица КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter, синхронизация с Объект обработки.НастройкиУниверсальногоФильтраПортфелейСКД при открытии и перед выполнением или записью параметров. В минимальном варианте фильтр только копируется в объект обработки и используется серверной логикой без отдельной формы отбора.

6. Подбор портфелей: как у контроля поручений (параметры контекста)

Базовая логика (типы клиентов, стратегии и т.д.) формирует массив кандидатов. Сужение по СКД делается не ручным пересечением в коде, а параметрами макета ДействующиеПортфели (см. 01_detailed_spec.md, п. 6.1 и 02_code_templates.md):

  • ОграничитьКандидатами — если Истина, в набор попадают только ссылки из массива кандидатов;
  • КандидатыПортфели — массив ссылок на портфели (при ограничении); при ОграничитьКандидатами = Ложь условие по списку в запросе не действует (отбор «по всем» в рамках базового запроса макета).

Дальше в компоновщик подмешиваются сохраненные пользователем НастройкиКомпоновкиДанных из хранилища; выполняется вывод в коллекцию; результат — массив ссылок Ссылка.

Обработка вызывает обертку (например ПрименитьУниверсальныйФильтрПортфелейСКД(МассивКандидатов)), внутри — Справочники.Портфели.ПортфелиПоНастройкеСКДСКонтекстом(..., Истина, МассивКандидатов) без передачи даты в схему.

Актуальные тексты модулей и формы — в 02_code_templates.md; единая нотация имён для вендора — в 07_vendor_spec_avancore.md (реквизит объекта НастройкиУниверсальногоФильтраПортфелейСКД, реквизит формы КомпоновщикУниверсальногоФильтраПортфелейСКД).

7. Контрольный чек-лист

  • Хранилище настроек СКД в справочнике настроек и в обработке.
  • Макет ДействующиеПортфели с параметрами ОграничитьКандидатами, КандидатыПортфели (без параметра даты дня для регламента).
  • Форма элемента настроек: компоновщик SettingsComposer, таблица ...Settings.Filter, ПриСозданииНаСервере / ПриЧтенииНаСервере / ПередЗаписьюНаСервере (как КонтрольПоручений).
  • Менеджер Портфели: одна функция выполнения СКД с контекстом; обработка подставляет массив кандидатов после штатного отбора.

Подробные шаги и примеры кода: 03_developer_guide.html, 02_code_templates.md.

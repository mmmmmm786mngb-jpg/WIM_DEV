IMAPPS-36274: фильтр портфелей по СКД

Краткая схема для разработчика: реквизиты, формы, макет, перенос в обработку, сужение списка портфелей.

2. Макет схемы СКД

  • Схема компоновки данных хранится как макет в справочнике Портфели (в реализации проекта: ДействующиеПортфели).
  • Текст запроса и поля набора должны позволять получить список портфелей, подходящих под отбор (как минимум колонка со ссылкой на портфель, например Ссылка).
  • При необходимости в схеме задаются параметры (например дата периода); при программной компоновке их нужно заполнять перед выполнением.

3. Форма элемента справочника настроек

Обязательно для штатной работы дерева отбора. В метаданных формы у реквизита НастройкиСКД тип должен быть КомпоновщикНастроекКомпоновкиДанных (в XML-выгрузке формы: dcsset:SettingsComposer). Тип НастройкиКомпоновкиДанных у реквизита формы для привязки таблицы ...Settings.Filter не подходит: дерево отбора на управляемой форме работает корректно только с компоновщиком.

Реквизит формы НастройкиСКД

  • Смысл: держать на форме редактируемое состояние фильтра СКД на время работы формы (объект компоновщика в памяти).
  • В базе данных сохраняется не реквизит формы, а Объект.ФильтрПортфелейСКД (хранилище с объектом НастройкиКомпоновкиДанных), получаемым через НастройкиСКД.ПолучитьНастройки().

Элемент формы

  • Таблица (дерево) с DataPath = НастройкиСКД.Settings.Filter, режим отображения отбора СКД (как в типовых формах с компоновщиком).
  • При открытии формы: инициализировать компоновщик по макету Портфели и при наличии загрузить сохраненные настройки из Объект.ФильтрПортфелейСКД.Получить().
  • Перед записью элемента справочника: Объект.ФильтрПортфелейСКД = Новый ХранилищеЗначения(НастройкиСКД.ПолучитьНастройки()) (или очистка, если фильтр сброшен).
  • Для уже записанных элементов после чтения из БД повторно вызывать ту же загрузку в компоновщик (событие чтения на сервере), чтобы данные из хранилища совпали с открытой формой.

Form.xml: подписка на серверные события (имена обработчиков замените на свои или используйте префикс расширения)

<Events>
    <Event name="OnReadAtServer">ИмяМодуля_ПриЧтенииНаСервере</Event>
    <Event name="BeforeWriteAtServer">ИмяМодуля_ПередЗаписьюНаСервере</Event>
</Events>

Дополнительно: подписка &После("ПриСозданииНаСервере") для нового элемента. Для записанных элементов нужна ПриЧтенииНаСервере с той же инициализацией (см. событие в Form.xml).

Module.bsl формы: инициализация компоновщика при открытии (вызывать из ПриСозданииНаСервере и ПриЧтенииНаСервере)

// Описание: Заполняет реквизит формы НастройкиСКД (КомпоновщикНастроекКомпоновкиДанных) из макета и Объект.ФильтрПортфелейСКД.
//
&НаСервере
Процедура ИнициализироватьНастройкиСКДКомпоновщикНаСервере()

    ИмяМакетаСКД = "ДействующиеПортфели";
    СхемаКомпоновкиДанных = Справочники.Портфели.ПолучитьМакет(ИмяМакетаСКД);
    КомпоновщикНастроек = Новый КомпоновщикНастроекКомпоновкиДанных;
    КомпоновщикНастроек.Инициализировать(Новый ИсточникДоступныхНастроекКомпоновкиДанных(СхемаКомпоновкиДанных));
    КомпоновщикНастроек.ЗагрузитьНастройки(СхемаКомпоновкиДанных.НастройкиПоУмолчанию);

    Если ТипЗнч(Объект.ФильтрПортфелейСКД) = Тип("ХранилищеЗначения") Тогда
        СохраненныеНастройки = Объект.ФильтрПортфелейСКД.Получить();
        Если ТипЗнч(СохраненныеНастройки) = Тип("НастройкиКомпоновкиДанных") Тогда
            КомпоновщикНастроек.ЗагрузитьНастройки(СохраненныеНастройки);
        КонецЕсли;
    КонецЕсли;

    НастройкиСКД = КомпоновщикНастроек;

4. Обработка: перенос фильтра из настроек

После выбора настройки регламентного закрытия (когда заполняются реквизиты обработки по настройке) скопировать:
Обработка.ФильтрПортфелейСКД = Настройка.ФильтрПортфелейСКД

Так пользователь видит на обработке тот же фильтр, что сохранен в элементе справочника настроек, без отдельного ввода.

5. Форма обработки (по необходимости)

Если требуется показывать или менять отбор на форме обработки, применяют ту же схему: реквизит НастройкиСКД (компоновщик), таблица НастройкиСКД.Settings.Filter, синхронизация с Объект обработки.ФильтрПортфелейСКД при открытии и перед выполнением или записью параметров. В минимальном варианте фильтр только копируется в объект обработки и используется серверной логикой без отдельной формы отбора.

6. Подбор портфелей: дополнительное сужение списка

Базовая логика (типы клиентов, стратегии и т.д.) формирует список портфелей в обработке. СКД-фильтр применяют к уже отобранным портфелям как пересечение множеств:

  1. Выполнить компоновку по макету в Портфели с параметрами периода и отбором из сохраненных НастройкиКомпоновкиДанных; получить массив ссылок портфелей, прошедших СКД.
  2. Взять текущий список портфелей обработки (после штатного заполнения).
  3. Оставить в списке только те портфели, которые есть и в СКД-результате (пересечение).

Если хранилище фильтра пустое или не задано, шаг с СКД не выполняют, список не сужают.

Модуль объекта обработки: после штатного ЗаполнитьСписокПортфелей — пересечение с результатом СКД

&НаСервере
&После("ЗаполнитьСписокПортфелей")
Процедура Расширение_ЗаполнитьСписокПортфелей(ДатаЗакрытия, СписокТипов = Неопределено, СписокПортфелейПоСтратегиям = Неопределено)

    Если ТипЗнч(ЭтотОбъект.ФильтрПортфелейСКД) <> Тип("ХранилищеЗначения") Тогда
        Возврат;
    КонецЕсли;

    НастройкиФильтра = ЭтотОбъект.ФильтрПортфелейСКД.Получить();
    Если ТипЗнч(НастройкиФильтра) <> Тип("НастройкиКомпоновкиДанных") Тогда
        Возврат;
    КонецЕсли;

    ОтобранныеПоСКД = ПолучитьПортфелиПоСКДФильтру(ДатаЗакрытия, НастройкиФильтра);
    Если Не ЗначениеЗаполнено(ОтобранныеПоСКД) Тогда
        ЭтотОбъект.СписокПортфелей.Очистить();
        Возврат;
    КонецЕсли;

    ИсходныйСписок = ЭтотОбъект.СписокПортфелей.ВыгрузитьЗначения();
    Пересечение = Новый Массив;

    Для каждого Портфель Из ИсходныйСписок Цикл
        Если ОтобранныеПоСКД.Найти(Портфель) <> Неопределено Тогда
            Пересечение.Добавить(Портфель);
        КонецЕсли;
    КонецЦикла;

    ЭтотОбъект.СписокПортфелей.Очистить();
    Для каждого Портфель Из Пересечение Цикл
        ЭтотОбъект.СписокПортфелей.Добавить(Портфель);
    КонецЦикла;

КонецПроцедуры

Выполнение СКД по макету Портфели: параметр периода + копирование отбора из сохраненных настроек (имя колонки — как в макете, чаще Ссылка)

&НаСервере
Функция ПолучитьПортфелиПоСКДФильтру(ДатаПериода, НастройкиФильтра)

    ТаблицаЗначений = Новый ТаблицаЗначений;
    СхемаКомпоновкиДанных = Справочники.Портфели.ПолучитьМакет("ДействующиеПортфели");

    КомпоновщикНастроек = Новый КомпоновщикНастроекКомпоновкиДанных;
    КомпоновщикНастроек.Инициализировать(Новый ИсточникДоступныхНастроекКомпоновкиДанных(СхемаКомпоновкиДанных));
    КомпоновщикНастроек.ЗагрузитьНастройки(СхемаКомпоновкиДанных.НастройкиПоУмолчанию);

    НастройкиКомпоновщика = КомпоновщикНастроек.ПолучитьНастройки();

    ПараметрДаты = НастройкиКомпоновщика.ПараметрыДанных.Элементы.Найти("ДатаПериода");
    Если ПараметрДаты <> Неопределено Тогда
        ПараметрДаты.Значение = ДатаПериода;
        ПараметрДаты.Использование = Истина;
    КонецЕсли;

    НастройкиКомпоновщика.Отбор.Элементы.Очистить();
    Для каждого ЭлементОтбора Из НастройкиФильтра.Отбор.Элементы Цикл
        Если Не ЭлементОтбора.Использование Тогда
            Продолжить;
        КонецЕсли;
        КопироватьЭлементОтбораСКД(НастройкиКомпоновщика.Отбор.Элементы, ЭлементОтбора);
    КонецЦикла;

    КомпоновщикМакета = Новый КомпоновщикМакетаКомпоновкиДанных;
    МакетКомпоновкиДанных = КомпоновщикМакета.Выполнить(
        СхемаКомпоновкиДанных,
        НастройкиКомпоновщика,
        ,
        ,
        Тип("ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений"));

    ПроцессорКомпоновкиДанных = Новый ПроцессорКомпоновкиДанных;
    ПроцессорКомпоновкиДанных.Инициализировать(МакетКомпоновкиДанных);

    ПроцессорВывода = Новый ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений;
    ПроцессорВывода.УстановитьОбъект(ТаблицаЗначений);
    ПроцессорВывода.Вывести(ПроцессорКомпоновкиДанных);

    Возврат ТаблицаЗначений.ВыгрузитьКолонку("Ссылка");

&НаСервере
Процедура КопироватьЭлементОтбораСКД(КоллекцияПриемник, ЭлементИсточник)

    Если ТипЗнч(ЭлементИсточник) = Тип("ЭлементОтбораКомпоновкиДанных") Тогда
        НовыйЭлемент = КоллекцияПриемник.Добавить(Тип("ЭлементОтбораКомпоновкиДанных"));
        НовыйЭлемент.Использование = ЭлементИсточник.Использование;
        НовыйЭлемент.ЛевоеЗначение = ЭлементИсточник.ЛевоеЗначение;
        НовыйЭлемент.ВидСравнения = ЭлементИсточник.ВидСравнения;
        НовыйЭлемент.ПравоеЗначение = ЭлементИсточник.ПравоеЗначение;
        Возврат;
    КонецЕсли;

    Если ТипЗнч(ЭлементИсточник) = Тип("ГруппаЭлементовОтбораКомпоновкиДанных") Тогда
        НоваяГруппа = КоллекцияПриемник.Добавить(Тип("ГруппаЭлементовОтбораКомпоновкиДанных"));
        НоваяГруппа.Использование = ЭлементИсточник.Использование;
        НоваяГруппа.ТипГруппы = ЭлементИсточник.ТипГруппы;
        Для каждого ВложенныйЭлемент Из ЭлементИсточник.Элементы Цикл
            КопироватьЭлементОтбораСКД(НоваяГруппа.Элементы, ВложенныйЭлемент);
        КонецЦикла;
    КонецЕсли;

КонецПроцедуры

Имя параметра ДатаПериода и поле Ссылка должны совпадать с макетом СКД. Процедуру копирования отбора переименуйте в одно имя с модулем обработки, чтобы не дублировать с формой.

7. Контрольный чек-лист

  • Реквизит ФильтрПортфелейСКД в справочнике настроек и в обработке.
  • Макет СКД в Портфели, единый источник схемы для формы настроек и для серверного выполнения в обработке.
  • Форма настроек: реквизит НастройкиСКД с типом КомпоновщикНастроекКомпоновкиДанных; загрузка из хранилища при открытии/после чтения; запись в ФильтрПортфелейСКД перед записью элемента.
  • Обработка: копирование фильтра из выбранной настройки; после заполнения списка портфелей — пересечение с результатом СКД.

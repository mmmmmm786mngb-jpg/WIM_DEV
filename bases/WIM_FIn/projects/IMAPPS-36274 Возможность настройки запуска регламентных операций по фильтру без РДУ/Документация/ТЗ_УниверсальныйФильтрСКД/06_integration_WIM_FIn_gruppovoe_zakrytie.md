# Пример встраивания универсального фильтра СКД в групповое закрытие (WIM_FIn)

## Где лежит код

Обработка **`ГрупповоеВыполнениеЗакрытияПериодов`** в выгрузке конфигурации **WIM_FIn** (каталог из `bases/WIM_FIn/source-path.txt`, типичный путь `...\WIM_FIn\src\cf\...`):

- `DataProcessors/ГрупповоеВыполнениеЗакрытияПериодов/Ext/ObjectModule.bsl`

В базе **Wim_PIF** (`bases/Wim_Pif/source-path.txt`) этой обработки в типовом дереве нет: закрытие ПИФ устроено иначе. Задача IMAPPS-36274 по смыслу относится к **ФИН**; пример ниже — для **WIM_FIn**.

## Уже есть: перенос настроек из справочника

Справочник **`НастройкиЗакрытияПортфелейПоРасписанию`**: при регламентном запуске вызывается `ЗаполнитьПериодыПортфелей(НастройкаРегулярногоЗакрытия)`, внутри — **`ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия`**.

Сейчас туда переносятся типы клиентов, стратегии, список портфелей из табличной части настройки и др. Сюда же нужно добавить перенос хранилища СКД (после добавления реквизита в метаданные обработки и справочника по `01_detailed_spec.md`). Отдельного булева «использовать фильтр» нет.

```bsl
    // сразу после загрузки стратегий / списка портфелей, например после строки с СоздаватьОбщиеОперации
    ЭтотОбъект.НастройкиУниверсальногоФильтраПортфелейСКД = НастройкаРегулярногоЗакрытия.НастройкиУниверсальногоФильтраПортфелейСКД;
```

Если обращение к реквизитам идёт через ссылку без чтения объекта, используйте `ОбщегоНазначения.ЗначенияРеквизитовОбъекта(НастройкаРегулярногоЗакрытия, "...")` по правилам вашей конфигурации.

## Функция-обёртка в модуле обработки

Полный текст — в **`02_code_templates.md`**, раздел 4.2: **`ПрименитьУниверсальныйФильтрПортфелейСКД(МассивПортфелейКандидатов)`** (дату дня в СКД не передают), внутри вызов **`Справочники.Портфели.ПортфелиПоНастройкеСКДСКонтекстом`** с **`ОграничитьКандидатами = Истина`** и переданным массивом.

Смысл: массив после **`ДействующиеПортфели`** сужается в СКД параметрами **`ОграничитьКандидатами` / `КандидатыПортфели`** в макете **`ДействующиеПортфели`** (см. `01_detailed_spec.md`, п. 6.1).

## Место 1: `ЗаполнитьСписокПортфелей`

Сейчас после отбора типов/стратегий вызывается:

```77:85:C:/1c/Cursor_1c/WORK/WIM_FIn/src/cf/DataProcessors/ГрупповоеВыполнениеЗакрытияПериодов/Ext/ObjectModule.bsl
	ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(ДатаЗакрытия, СписокТипов, СписокПортфелейПоСтратегиям);
		
	ОбщегоНазначенияДУ.ДобавитьВЛогОтладки(ДействующиеПортфели, "Обработка.ГрупповоеВыполнениеЗакрытияПериодов.ЗаполнитьСписокПортфелей()");
	
	Если ЗначениеЗаполнено(ДействующиеПортфели) Тогда
		Для Каждого ПортфельСсылка Из ДействующиеПортфели Цикл			
			СписокПортфелей.Добавить(ПортфельСсылка);
		КонецЦикла;
	КонецЕсли;
```

**Доработка:** между вызовом `ДействующиеПортфели` и логом (или сразу после вызова) вставить:

```bsl
	ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

Так фильтр применится во всех сценариях, где заполняется **`СписокПортфелей`** через эту процедуру (в т.ч. **`ЗаполнитьПериодыПортфелейПоПериоду`**).

## Место 2: `ЗаполнитьПериодыПортфелейПоГраницам`

Здесь для каждого **`День`** строится массив **`ДействующиеПортфели`** и проверяется вхождение портфеля из границы:

```738:741:C:/1c/Cursor_1c/WORK/WIM_FIn/src/cf/DataProcessors/ГрупповоеВыполнениеЗакрытияПериодов/Ext/ObjectModule.bsl
		Если ЭтотОбъект.ПоВсемДействующимПортфелям Тогда
			СписокПортфелейПоСтратегиям = СписокПортфелейПоСтратегиям(СтратегииДУ, День);
			ДействующиеПортфели = Справочники.Портфели.ДействующиеПортфели(День, ТипыКлиентовДУ, СписокПортфелейПоСтратегиям);
		КонецЕсли; 		
```

**Доработка:** сразу после присваивания `ДействующиеПортфели` добавить:

```bsl
			ДействующиеПортфели = ПрименитьУниверсальныйФильтрПортфелейСКД(ДействующиеПортфели);
```

## Интерактивный режим и форма обработки

Общая схема передачи полей с формы в исполнение в WIM_FIn: **`РеквизитФормыВЗначение("Объект")`** / **`ТекущийОбъект()`** → вызовы методов модуля объекта → при необходимости **`ЗначениеВРеквизитФормы(ТекущийОбъект(), "Объект")`**. Подробно — **`07_vendor_spec_avancore.md`**, п. **6.0**.

Если пользователь открывает обработку вручную и не вызывает `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия`, реквизит объекта **`НастройкиУниверсальногоФильтраПортфелейСКД`** остаётся пустым, пока его не заполнят с формы.

На **форме** обработки нужен реквизит **`КомпоновщикУниверсальногоФильтраПортфелейСКД`** и дерево отбора (как в ТЗ для справочника). Изменения в дереве сами по себе **не** обновляют `ЭтотОбъект` в модуле объекта: перед командами, которые запускают расчёт списков портфелей, на сервере вызывают синхронизацию компоновщика в `Объект.НастройкиУниверсальногоФильтраПортфелейСКД` (см. **`02_code_templates.md`**, п. **4.4**). Иначе `ПрименитьУниверсальныйФильтрПортфелейСКД` будет использовать устаревшее или пустое хранилище.

При запуске из регламента форма не участвует: хранилище уже перенесено в объект в `ЗаполнитьРеквизитыПоНастройкеРегулярногоЗакрытия`.

## Зависимости перед компиляцией

1. Реквизиты обработки и справочника настроек по ТЗ.
2. Параметры в макете **`ДействующиеПортфели`** и доработка запроса набора данных.
3. Экспортная функция **`ПортфелиПоНастройкеСКДСКонтекстом`** в **`Справочники.Портфели`** (`02_code_templates.md`, раздел 3).

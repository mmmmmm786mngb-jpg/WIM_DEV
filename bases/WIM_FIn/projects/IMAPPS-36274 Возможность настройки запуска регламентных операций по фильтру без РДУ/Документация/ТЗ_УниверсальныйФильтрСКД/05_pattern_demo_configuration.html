IMAPPS-36274: блок фильтра СКД по образцу демо конфигурации

Как перенести проверенные приемы из внутреннего эталона (демо конфигурация) в настройки регламентного закрытия и обработку.

1. Откуда брать образец

В демо конфигурации (внутренний эталон, не поставка) смотреть справочник с отбором через СКД:

  • Объект справочника — реквизит НастройкиСКД (ХранилищеЗначения), отдельный макет схемы компоновки для отбора.
  • Форма элемента — реквизит формы КомпоновщикСхемы с типом dcsset:SettingsComposer, таблица КомпоновщикСхемы.Settings.Filter.
  • Модуль менеджера справочника — выполнение СКД по сохраненным настройкам и параметру контекста (УстановитьПараметрыСКД или аналог).

3. Метаданные (как в демо)

  • В справочнике настроек регламента добавить реквизит НастройкиУниверсальногоФильтраПортфелейСКД, тип ХранилищеЗначения (как НастройкиСКД в демо).
  • В обработке закрытия — такой же реквизит для рабочей копии; копировать из выбранной настройки при открытии (см. 04_developer_brief.html).

4. Форма элемента настройки: XML

Повторить структуру формы элемента эталонного справочника в демо, заменив имя реквизита компоновщика на КомпоновщикУниверсальногоФильтраПортфелейСКД (IMAPPS-36274):

  • Реквизит формы: тип dcsset:SettingsComposer (в конфигураторе — КомпоновщикНастроекКомпоновкиДанных).
  • Таблица (дерево): DataPath = КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter, представление дерева, режим настроек СКД.
  • События: минимум OnCreateAtServer, BeforeWriteAtServer. Для расширения и повторного чтения объекта — еще OnReadAtServer с той же инициализацией, что и при создании (в демо на форме эталона подписан только при создании; для надежности в расширении лучше дублировать).
Идея из демо: схему СКД помещают во временное хранилище и передают в ИсточникДоступныхНастроекКомпоновкиДанных по адресу. Это устойчиво для тонкого клиента и совпадает по смыслу с прямой инициализацией из макета в брифе — выберите один стиль в проекте и придерживайтесь его везде.

6. Выполнение СКД в обработке (логика менеджера в демо)

В эталонном справочнике демо после чтения настроек из ссылки выставляют внешний параметр и строят макет компоновки. Для портфелей по IMAPPS-36274: загрузить НастройкиКомпоновкиДанных из хранилища, установить параметры ОграничитьКандидатами и КандидатыПортфели (дату дня в схему СКД не подставляют — кандидаты уже отобраны на нужную дату), затем КомпоновщикМакета + ПроцессорКомпоновкиДанных + вывод в коллекцию.

УстановитьПараметрСКД(НастройкиКомпоновщика, "ОграничитьКандидатами", Истина);
УстановитьПараметрСКД(НастройкиКомпоновщика, "КандидатыПортфели", МассивКандидатов);

Имя процедуры и сигнатура — как в 02_code_templates.md; при наличии обёртки в конфигурации вызывать её.

Имена параметров и полей результата должны совпадать с макетом ДействующиеПортфели (часто колонка Ссылка).

Дальше — пересечение с уже заполненным списком портфелей обработки, как в разделе 6 04_developer_brief.html.

7. Дополнительные идеи из демо

  • Копирование элемента: ветка Параметры.ЗначениеКопирования при загрузке настроек — чтобы фильтр переносился вместе с новой настройкой регламента.
  • Защита от ошибок в данных СКД: в менеджере эталонного справочника настройки обрабатываются в Попытка ... Исключение в цикле; для регламента имеет смысл логировать сбой и не ронять весь сеанс закрытия.
  • Единый макет: как в демо макет отбора лежит на справочнике-источнике, макет отбора портфелей удобно держать у Портфели, чтобы форма и сервер брали одну схему.

8. Чек-лист

  1. НастройкиУниверсальногоФильтраПортфелейСКД в справочнике настроек и в обработке (ХранилищеЗначения).
  2. Реквизит формы — КомпоновщикНастроекКомпоновкиДанных (SettingsComposer в XML).
  3. Таблица отбора привязана к КомпоновщикУниверсальногоФильтраПортфелейСКД.Settings.Filter.
  4. Инициализация: макет Портфели.ДействующиеПортфели, при необходимости ВХ по образцу демо.
  5. Запись в хранилище объекта перед записью элемента; повторная инициализация после чтения при работе через расширение.
  6. Серверное выполнение СКД с установкой параметров периода и пересечением списка портфелей.

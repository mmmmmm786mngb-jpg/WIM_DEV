IMAPPS-36274 - Список измененных объектов для вендора Avancore

Задача: IMAPPS-36274 "Возможность настройки запуска регламентных операций по фильтру без РДУ"

Источник: выгрузка расширения FP в каталоге проекта WIM_DEV; дополнительно постановка в Jira.txt (компонент AvancoreFinance).

Расширение: имя в метаданных fp, префикс процедур fp_, режим совместимости Version8_3_25 (см. FP/Configuration.xml).

Цель доработки: добавить гибкий отбор портфелей для "Группового выполнения закрытия периода", чтобы снизить зависимость только от типов клиентов и ручных списков (в т.ч. для сценариев с РДУ).

Иерархическая схема: imapps36274_tree_changed_objects_fp.html (дерево объектов, по аналогии с rev).

4. Примечание для Avancore

Важно: в XML макета СКД параметр в тексте запроса отображается как &amp;СписокПортфелей, что является стандартным XML-экранированием символа &. В исполняемом тексте запроса 1С это параметр &СписокПортфелей.

Проверка полноты: перехватчики и имена процедур в ObjectModule.bsl обработки, сигнатуры ПолучитьПортфелиПоОтборуСКД / УстановитьПараметрыСКД в менеджере Портфели, подписки на события в Form.xml — сверены с выгрузкой FP в репозитории WIM_DEV (проект IMAPPS-36274).

Документ актуализирован по составу расширения FP в каталоге проекта; при расхождении с вашей веткой ориентируйтесь на фактическую выгрузку CFE.

IMAPPS-36274 - Список измененных объектов для вендора Avancore

Задача: IMAPPS-36274 "Возможность настройки запуска регламентных операций по фильтру без РДУ"

Источник: Jira.txt (компонент AvancoreFinance)

Расширение с доработками: FP

Цель доработки: добавить гибкий отбор портфелей для "Группового выполнения закрытия периода", чтобы исключить зависимость от выбора только по типам/ручным спискам.

  • Логика обработки отбора по СКД в объекте Обработка.ГрупповоеВыполнениеЗакрытияПериодов.
  • Новая функция фильтрации в Справочник.Портфели + макет МакетОтбораПортфелей.

4. Примечание для Avancore

Важно: в XML макета СКД параметр в тексте запроса отображается как &amp;СписокПортфелей, что является стандартным XML-экранированием символа &. В исполняемом тексте запроса 1С это параметр &СписокПортфелей.

h2. IMAPPS-36274 | Техническая реализация для вендора (комментарий JIRA 8.13, Wiki)

{panel:title=Как использовать|borderStyle=solid|borderColor=#c5d4e0|titleBGColor=#e8eef5|bgColor=#ffffff}
Скопируйте текст *ниже этой панели* (с раздела "Назначение") в комментарий задачи в режиме *Wiki markup*. Структура ориентирована на разработчика/аналитика вендора (AvancoreFinance).
{panel}

----

h3. Назначение

Перенос в типовую поставку (или согласованное расширение) логики из референс-расширения *FP*: дополнительный отбор портфелей по пользовательской СКД поверх штатного списка, настройка хранится в элементе *Справочник.НастройкиЗакрытияПортфелейПоРасписанию*.

{info}*Принцип:* штатный вызов *Справочники.Портфели.ДействующиеПортфели(...)* не заменяется; после него вызывается сужение через *ПолучитьПортфелиПоОтборуСКД(...)*. При пустых настройках СКД возвращается исходный массив без изменений.{info}

----

h3. 1. Метаданные: справочник НастройкиЗакрытияПортфелейПоРасписанию

* *Реквизит объекта:* {{НастройкиОтбораПортфелейСКД}}, тип *ХранилищеЗначения* (сериализация {{НастройкиКомпоновкиДанных}} компоновщика).
* *Форма:* {{Справочник.НастройкиЗакрытияПортфелейПоРасписанию.Форма.ФормаЭлемента}} (в референсе FP заимствована и расширена).
* *События формы (callType After):*
** {{OnCreateAtServer}} -> {{fp_ПриСозданииНаСервереПосле}}
** {{BeforeWriteAtServer}} -> {{fp_ПередЗаписьюНаСервереПосле}}
* *Логика модуля формы (сервер):*
** получить макет: {{Справочники.Портфели.ПолучитьМакет("МакетОтбораПортфелей")}};
** поместить схему во временное хранилище, {{ИсточникДоступныхНастроекКомпоновкиДанных}}, инициализация реквизита формы-компоновщика (в референсе: {{КомпоновщикНастроекОтборПортфелей}});
** при создании: загрузить настройки из {{Объект.НастройкиОтбораПортфелейСКД.Получить()}} либо из {{СхемаКомпоновки.НастройкиПоУмолчанию}}; учесть копирование элемента ({{Параметры.ЗначениеКопирования}});
** перед записью на сервере: {{ТекущийОбъект.НастройкиОтбораПортфелейСКД = Новый ХранилищеЗначения(КомпоновщикНастроекОтборПортфелей.Настройки)}} (имя компоновщика в EDT должно совпадать с формой).

----

h3. 2. Метаданные: справочник Портфели

* *Модуль менеджера* ({{Ext.ManagerModule}}):
** экспортная функция {{ПолучитьПортфелиПоОтборуСКД(НастройкаЗакрытияПортфелейПоРасписанию, ИсходныйСписокПортфелей)}};
** процедура {{УстановитьПараметрыСКД(Настройки, ИмяПараметра, Значение)}} -- установка {{Настройки.ПараметрыДанных.Элементы.Найти(ИмяПараметра)}} с {{Использование = Истина}}.
* *Алгоритм {{ПолучитьПортфелиПоОтборуСКД}}:*
** выход без изменений, если не заполнена ссылка настройки, пуст {{ИсходныйСписокПортфелей}}, пусто хранилище {{НастройкиОтбораПортфелейСКД}}, либо в восстановленных настройках нет элементов отбора;
** схема: {{ПолучитьМакет("МакетОтбораПортфелей")}};
** параметр данных СКД: {{СписокПортфелей}} <- исходный массив ссылок;
** выполнение: {{КомпоновщикМакетаКомпоновкиДанных}} + {{ПроцессорКомпоновкиДанных}} + {{ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений}};
** результат: {{ТаблицаЗначений.ВыгрузитьКолонку("Ссылка")}}.
* *Макет СКД:* {{Справочник.Портфели.МакетОтбораПортфелей}} (в XML выгрузке: {{Templates/МакетОтбораПортфелей}}).
** набор данных: запрос к {{Справочник.Портфели}} с условием {{Ссылка В(&СписокПортфелей)}} (в файле Template.xml параметр в тексте запроса экранируется как {{&amp;СписокПортфелей}} -- это норма для XML).

h3. 4. Состав расширения FP (для сверки с поставкой)

Заявленные в {{Configuration.xml}} дочерние объекты (кроме языка): {{StyleItem.ЦветГиперссылки}}, {{CommonPicture.Плюс}}, {{Catalog.НастройкиЗакрытияПортфелейПоРасписанию}}, {{Catalog.ВидыОперацийЗакрытияПериода}}, {{Catalog.Портфели}}, {{DataProcessor.ГрупповоеВыполнениеЗакрытияПериодов}}.

* *С изменением логики/структуры:* настройки расписания (реквизит + форма), портфели (менеджер + макет), групповое закрытие (модуль объекта).
* *Без изменения кода в референсе:* {{ВидыОперацийЗакрытияПериода}}, {{Плюс}}, {{ЦветГиперссылки}} -- техническое включение в состав расширения.

h3. 6. Критерии приемки (технические)

# Сохранение/чтение {{НастройкиОтбораПортфелейСКД}} на форме элемента настройки; корректное поведение при копировании элемента.
# После {{ДействующиеПортфели}} во всех задействованных ветках обработки итоговый массив согласован с СКД при непустом отборе.
# При пустом отборе или пустом хранилище результат совпадает с вызовом только {{ДействующиеПортфели}}.
# Регламентный сценарий: объект обработки получает ссылку на настройку и применяет тот же фильтр без открытия формы.
# Совместимость режима платформы расширения: в референсе {{ConfigurationExtensionCompatibilityMode}} / {{CompatibilityMode}} -- 8.3.25 (проверить на целевой ветке поставки).

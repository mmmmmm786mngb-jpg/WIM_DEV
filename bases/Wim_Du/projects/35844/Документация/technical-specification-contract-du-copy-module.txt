h1. Техническое задание на разработку модуля «Копирование договоров ДУ по шаблону»

*Версия документа:* 1.0
*База:* Wim_Du
*Проект:* 35844
*Внешняя обработка:* ГенераторДоговоровДУ

----

h2. 1. Общие сведения

h3. 1.1 Назначение

Разработать внешнюю обработку (управляемая форма) для массового создания копий договоров доверительного управления (ДоговорДУ) по выбранному договору-шаблону с автоматическим копированием связанных объектов: элементов справочников *АналитическиеСчета*, *ДоговорыКонтрагентов* и *ТорговыеКоды*, документов *Учетная политика* и *Сделка* (вид операции Поступление).

h3. 1.2 Цели

* _Основная цель:_ обеспечить возможность массового создания тестовых договоров ДУ для нагрузочного тестирования; созданные копии должны быть пригодны для расчёта регламентных операций и для генерации и исполнения Сделок.
* Сократить ручной ввод при создании серии однотипных договоров ДУ (шаблон + связанные объекты копируются автоматически).
* Обеспечить корректную подстановку нового кода договора во все копируемые объекты (в т.ч. в строковых реквизитах через замену кода шаблона).
* Сохранить связь скопированных АналитическихСчетов с корректными скопированными ДоговорамиКонтрагентов (Владелец ДоговорыКонтрагентов — Контрагент; поиск шаблонных договоров контрагентов выполнять через АналитическиеСчета шаблонного ДоговораДУ).

h3. 1.3 Используемые объекты конфигурации

||Объект||Тип||Назначение в ТЗ||
|ДоговорДУ|Справочник|Шаблон и создаваемые копии|
|ДоговорыКонтрагентов|Справочник (Владелец — Контрагенты)|Копируются по связи из АналитическихСчетов|
|АналитическиеСчета|Справочник|Копируются по ДоговорДУ шаблона|
|ТорговыеКоды|Справочник|Копируются по ДоговорДУ шаблона; подстановка нового ДоговораДУ, ДоговорБрокера из соответствия, АналитическийСчетДС/ЦБ по виду; замена цифр кода в ТорговыйКод и Наименование|
|УчетнаяПолитика|Документ|Копируется по шаблону для каждого нового ДоговораДУ|
|Сделка|Документ|Копируются проведённые документы с видом операции Поступление по ДоговоруДУ шаблона|

----

h2. 2. Термины и определения

* *Договор-шаблон* — выбранный пользователем элемент справочника ДоговорДУ, по данным которого создаются копии.
* *Код шаблона* — значение реквизита Код договора-шаблона; используется для замены в строковых полях копий на новый код.
* *Новый код договора* — строка вида «Префикс + (900000 + N)», где N — порядковый номер копии в цикле (например, ТТ900001, ТТ900002).
* *СубъектМестаХранения* — реквизит справочника АналитическиеСчета (тип Контрагенты); совпадает с Владельцем элемента ДоговорыКонтрагентов для договоров, используемых в этих счетах.

----

h2. 3. Требования к реквизитам формы (объект формы)

||Реквизит||Тип||Обязательность||Описание||
|ДоговорДУ_Шаблон|СправочникСсылка.ДоговорДУ|Да|Договор ДУ, используемый как шаблон|
|УчетнаяПолитика_Шаблон|ДокументСсылка.УчетнаяПолитика|Да*|Документ Учетная политика по договору-шаблону (*подставляется при изменении ДоговорДУ_Шаблон)|
|КоличествоКопий|Число|Да|Количество создаваемых копий (если не задано или ≤ 0 — считать 1)|
|НачальныйНомерКопии|Число|Нет|Смещение нумерации копий (0 = с первой, иначе N-я копия получает номер 900000 + НачальныйНомерКопии + (1..КоличествоКопий))|
|Префикс|Строка|Нет|Префикс кода новых договоров (по умолчанию при открытии формы — «ТТ»)|
|ДатаУстановкиДоговоров|Дата|Нет|Дата создания документов и период для периодических регистров (если не задана — текущая дата)|

----

h2. 4. Функциональные требования

h3. 4.1 Поведение при открытии формы

* При создании на сервере установить значение реквизита *Префикс* = «ТТ», если иное не требуется настройками.

h3. 4.2 Поведение при изменении договора-шаблона

* При изменении реквизита *ДоговорДУ_Шаблон* выполнить поиск последнего по дате проведённого документа *Учетная политика* с реквизитом ДоговорДУ = ДоговорДУ_Шаблон и датой ≤ текущей даты.
* Найденный документ подставить в реквизит формы *УчетнаяПолитика_Шаблон*. Если не найден — очистить УчетнаяПолитика_Шаблон.

h3. 4.3 Команда «Копировать по шаблону»

По нажатию кнопки (команды) выполнять на сервере процедуру копирования. Последовательность операций для *каждой* из КоличествоКопий итераций:

# *Создание нового ДоговораДУ*
# *Копирование АналитическихСчетов и ДоговоровКонтрагентов* (см. п. 5.2)
# *Копирование элементов справочника ТорговыеКоды* (см. п. 5.4)
# *Создание документа Учетная политика* по шаблону
# *Копирование документов Сделка* (вид операции Поступление)

Перед циклом необходимо: проверить заполненность ДоговорДУ_Шаблон и УчетнаяПолитика_Шаблон, получить объекты шаблонного ДоговораДУ и документа Учетная политика, вычислить ДатаУстановки и КодШаблона (значение Код шаблонного договора). В цикле каждую итерацию оборачивать в Попытка/Исключение с сообщением пользователю об ошибке; после цикла вывести итоговое сообщение (количество созданных договоров и документов Учетная политика).

----

h2. 5. Требования к копированию объектов

h3. 5.1 Создание нового элемента ДоговорДУ

* Создать элемент справочника ДоговорДУ. Скопировать из шаблона все реквизиты, *исключая*: Владелец, Ссылка, Код, ПометкаУдаления, Родитель.
* *Код* нового элемента задать: {{Префикс + Формат(900000 + НомерКопии, "ЧГ=0")}} (например, ТТ900001).
* *Наименование*: выполнить подстановку кода — в наименовании шаблона заменить строку КодШаблона на НовыйКод (СтрЗаменить).
* *Префикс* (реквизит справочника): если в шаблоне заполнен — в его значении выполнить СтрЗаменить(КодШаблона, НовыйКод).
* Записать элемент. Передать ссылку на созданный договор в последующие шаги.

h3. 5.2 Копирование АналитическихСчетов и ДоговоровКонтрагентов

*Важно:* Справочник ДоговорыКонтрагентов имеет *Владельца* типа Контрагенты. Искать договоры контрагентов для копирования не по ДоговоруДУ напрямую, а через АналитическиеСчета шаблонного ДоговораДУ: в каждом таком счёте хранятся ДоговорКонтрагента и СубъектМестаХранения (контрагент = Владелец договора контрагента).

*Шаг 1 — Получение списка АналитическихСчетов шаблона*

* Запросом выбрать из Справочник.АналитическиеСчета все элементы, у которых ДоговорДУ = ДоговорДУ_Шаблон и ПометкаУдаления = Ложь.
* В выборку включить поля: Ссылка, ДоговорКонтрагента, СубъектМестаХранения.

*Шаг 2 — Копирование уникальных ДоговоровКонтрагентов*

* Обойти выборку; для каждого уникального значения ДоговорКонтрагента (если пусто — пропустить):
** Загрузить объект шаблонного ДоговорыКонтрагентов.
** Создать новый элемент ДоговорыКонтрагентов.
** Заполнить значения из шаблона, *исключая*: Код, Ссылка, ДоговорДУ. Владелец копируется как есть (остаётся тот же контрагент).
** Установить ДоговорДУ = НовыйДоговор (созданный на шаге 4.3.1).
** В строковых реквизитах выполнить замену кода шаблона на новый код (СтрЗаменить(КодШаблона, НовыйКод)) для: Наименование, НомерДоговора, НомерСчетаДЕПО, НаименованиеРегл, Комментарий.
** Записать элемент. Сохранить в структуру соответствия: «старая ссылка ДоговорКонтрагента → новая ссылка» (для использования при копировании счетов).

*Шаг 3 — Копирование АналитическихСчетов*

* Обойти выборку АналитическихСчетов (повторно).
** Для каждой строки загрузить объект АналитическогоСчета по Ссылке.
** Создать новый элемент АналитическиеСчета. Заполнить из шаблона, *исключая*: Код, Ссылка, ДоговорДУ, ДоговорКонтрагента.
** Установить ДоговорДУ = НовыйДоговор.
** Установить ДоговорКонтрагента = значение из соответствия по шаблонному ДоговорКонтрагента.
** В строковых реквизитах выполнить СтрЗаменить(КодШаблона, НовыйКод) для: Наименование, НомерСчета, НаименованиеРаздела, Служебный, Комментарий.
** Записать элемент.

Ошибки при записи отдельных ДоговоровКонтрагентов или АналитическихСчетов обрабатывать (Попытка/Исключение), сообщать пользователю, не прерывая цикл по остальным элементам.

h3. 5.3 Создание документа Учетная политика

* Создать документ Учетная политика. Скопировать реквизиты и табличные части из шаблона УчетнаяПолитика_Шаблон, исключая Номер, Ссылка.
* Установить ДоговорДУ = НовыйДоговор.
* Записать документ (Запись). Затем выполнить проведение (Попытка/Исключение); при ошибке проведения сообщить пользователю, ссылку на созданный документ вернуть.

h3. 5.4 Копирование элементов справочника ТорговыеКоды

* Выполняется *после* копирования АналитическихСчетов (в рамках той же процедуры), чтобы были созданы новые счета по новому ДоговоруДУ.
* Запросом выбрать все элементы Справочник.ТорговыеКоды, у которых ДоговорДУ = ДоговорДУ_Шаблон и ПометкаУдаления = Ложь.
* Для подстановки АналитическийСчетДС и АналитическийСчетЦБ запросить по новому ДоговоруДУ первые (ПЕРВЫЕ 1) элементы АналитическиеСчета с ВидАналитическогоСчета = Перечисления.ВидыАналитическогоСчета.ДенежныеСредства и ЦенныеБумаги соответственно. При отсутствии счёта — подставлять пустую ссылку.
* Для каждого отобранного ТорговогоКода:
** Создать новый элемент ТорговыеКоды. Заполнить из шаблона, *исключая*: Код, Ссылка, ДоговорДУ, ДоговорБрокера, АналитическийСчетДС, АналитическийСчетЦБ.
** Установить ДоговорДУ = НовыйДоговор.
** Установить ДоговорБрокера = значение из соответствия «старый ДоговорКонтрагента → новый» (то же, что для АналитическихСчетов).
** Установить АналитическийСчетДС и АналитическийСчетЦБ — полученные по запросу ссылки на новые АналитическиеСчета по НовыйДоговор и виду.
** В реквизитах *ТорговыйКод* и *Наименование*: заменить только *цифровую часть* кода договора на цифры нового кода (из кода шаблона извлечь подстроку из цифр, из нового кода — аналогично; в значении реквизита выполнить СтрЗаменить(ЦифрыШаблона, ЦифрыНовые)). Если цифровая часть шаблона пуста — замену не выполнять.
** Табличная часть СписокКодов: скопировать; в каждой строке в реквизите ТорговыйКод выполнить ту же замену цифр кода договора.
** Записать элемент (Попытка/Исключение с сообщением пользователю при ошибке).

h3. 5.5 Копирование документов Сделка

* Запросом выбрать до 10 проведённых документов Сделка, у которых ДоговорДУ = ДоговорДУ_Шаблон и ВидОперации = Поступление, упорядоченных по Дате.
* Для каждого отобранного документа: создать новый документ Сделка, скопировать реквизиты и табличные части (исключая Номер, Ссылка), установить ДоговорДУ = НовыйДоговор. Записать и провести (при ошибке проведения — сообщить пользователю).

----

h2. 6. Проверки перед выполнением копирования

* Если не заполнен ДоговорДУ_Шаблон — вывести сообщение «Укажите договор ДУ — шаблон.» и завершить выполнение.
* Если не заполнен УчетнаяПолитика_Шаблон — вывести сообщение о том, что по выбранному договору не найден документ Учетная политика, и предложить выбрать договор с проведённой учетной политикой или указать шаблон.
* Если не удалось прочитать объект договора-шаблона или объект документа Учетная политика — вывести соответствующее сообщение и завершить выполнение.

----

h2. 7. Нефункциональные требования

* Код модуля формы — на встроенном языке 1С (BSL), управляемое приложение. Разделение контекста: клиент (вызов команды) и сервер (вся логика копирования, запросы, запись объектов).
* Запросы выполнять с использованием параметров, без конкатенации строк (безопасность от внедрения).
* Стиль кода: PascalCase для переменных и процедур, ключевые слова в стандартном регистре 1С (Не, И, Или, Тогда, КонецЕсли и т.д.), отступы 4 пробела.
* Экспортные и ключевые процедуры/функции должны иметь документирующие комментарии (Описание, Параметры, Возвращаемое значение для функций).

----

h2. 8. Критерии приёмки

* Пользователь может указать договор ДУ в качестве шаблона; при этом автоматически подставляется последний проведённый документ Учетная политика по этому договору.
* По команде создаётся заданное количество копий договора ДУ с кодами вида Префикс + (900000 + N).
* Для каждой копии создаются и привязываются к новому ДоговоруДУ: копии АналитическихСчетов шаблона с предварительным копированием связанных ДоговоровКонтрагентов (с тем же Владельцем-контрагентом); копии элементов ТорговыеКоды с подстановкой нового ДоговораДУ, ДоговорБрокера из соответствия и новых АналитическихСчетов (ДС/ЦБ по виду), с заменой только цифровой части кода в ТорговыйКод и Наименование (и в СписокКодов.ТорговыйКод); документ Учетная политика; копии проведённых Сделок с видом операции Поступление (до 10 шт.).
* Во всех копируемых объектах в указанных строковых реквизитах выполняется замена кода: полная замена кода шаблона на новый код (СтрЗаменить) или замена только цифровой части кода (для ТорговыеКоды — ТорговыйКод, Наименование, строки СписокКодов).
* При ошибках записи/проведения отдельных объектов пользователю выводятся сообщения, общий цикл копирования не прерывается (обработка по итерациям и внутри — по элементам).
* После выполнения выводится итоговое сообщение о количестве созданных договоров и документов Учетная политика.

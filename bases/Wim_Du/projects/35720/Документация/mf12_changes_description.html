1. Основание и цель доработки

Основание доработки - требования из ТЗ вознагр. СК ТИТ.docx и дополнительные требования пользователя.

Цель доработки - обеспечить корректный расчет MF12 с учетом даты ввода средств в ДУ и пропорционального расчета за неполный квартал.

Источники требований (базовые):
  • вознагр. СК ТИТ.docx - "Плата за управление составляет 500 000 руб. за календарный год. Взимается ежеквартально 125 000 руб."
  • вознагр. СК ТИТ.docx - при досрочном прекращении договора и неполном квартале расчет пропорционален сроку управления в отчетном квартале.
  • drafts/аа - уточнение: MF не должен считать только от начала года, нужно учитывать дату ввода в ДУ и неполный квартал при расторжении.

3. Основные изменения

  • Добавлен новый реквизит обработки НачалоПоДатеВводаДУ (тип Булево).
  • На форме настроек добавлен флажок НачалоПоДатеВводаДУ.
  • Настройка включена в чтение, сохранение и передачу структуры настроек алгоритма.
  • Пересчитаны границы периода: при включенном флаге начало расчета ограничивается датой ввода средств в ДУ.
  • Количество дней в расчете переведено на типовой подход базы: РазностьДатВКалендарныхДнях(...) + 1.
  • Версия обработки повышена до 1.01.

4. Логика расчета периода

До изменения

  • Начало периода брали из периодичности, без опции ограничения по дате ввода средств в ДУ.
  • Подсчет дней выполнялся по старой формуле, которая давала спорный результат при времени в датах.

После изменения

  • Если НачалоПоДатеВводаДУ = Истина, начало расчета = максимум из расчетного начала периода и даты ввода средств в ДУ.
  • Количество дней:
    • ВсегоДней = КалендарныеГрафикиПовтИсп.РазностьДатВКалендарныхДнях(ДатаНачала, ДатаОкончания) + 1
    • ПрошлоДней = КалендарныеГрафикиПовтИсп.РазностьДатВКалендарныхДнях(НачалоПериода, КонецПериода) + 1
Для одинаковых дат в пределах одного календарного дня результат по дням будет 1, что соответствует включительному подсчету.

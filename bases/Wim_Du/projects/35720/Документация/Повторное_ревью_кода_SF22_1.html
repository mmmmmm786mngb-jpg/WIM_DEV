4. Предложенная реализация исправления (по стандартам 1С) — реализовано

4.0. Режим процентов (реализовано)

В таблице и в коде используется режим процентов: Ставка (6, 5.25, 5). Формула: (П015 - MF) * ПроцентВознаграждения / 100. П021 и СтавкаВознаграждения передаются без * 100.

    ВызватьИсключение "Коэффициент должен быть задан долей: 0.06, 0.0525, 0.05";

4.1. Цель доработки (достигнута)

  • Зафиксировать единый формат хранения коэффициента в таблице: проценты (6, 5.25, 5).
  • Устранить неоднозначность в расчетах и исключить риск завышения суммы в 100 раз.
  • Сохранить совместимость с текущей печатной формой и структурой возврата РезультатРасчета().

4.2. Нормативное правило для кода

Правило: в ТаблицаКоэффициентов.Ставка хранятся проценты (6, 5.25, 5), не доли.

  • Корректные значения: 6, 5.25, 5.
  • Некорректные значения: 0.06, 0.0525, 0.05.

Формула расчета суммы должна всегда содержать деление на 100.

4.3. Изменения в модуле объекта (выполнены)

Функция: РезультатРасчета()

  1. Оставить выбор коэффициента из таблицы без преобразования (берем проценты как есть).
  2. Исправить расчет суммы SF: умножать на (КоэффициентВознаграждения / 100).
  3. В показатель П021 и в СтавкаВознаграждения передавать исходный процент без *100.

Рекомендуемый фрагмент кода 1С:

// Описание: расчет суммы SF при хранении коэффициента в процентах (6, 5.25, 5)
РачетнаяВеличинаВознаграждения = Окр(
    (СтруктураРасчетаПриростРСА.ПриростРСА - СуммаЗаУправление) * (КоэффициентВознаграждения / 100),
    2
);

ИтоговыйРасчет = ?(
    СтруктураРасчетаПриростРСА.ПриростРСА - СуммаЗаУправление > 0,

    0
);
ИтоговыйРасчет = Окр(ИтоговыйРасчет, 2);

// Ставка остается в процентах без дополнительного *100
ПоказателиСправки.П021 = КоэффициентВознаграждения;
СтруктураВозврата.СтавкаВознаграждения = КоэффициентВознаграждения;

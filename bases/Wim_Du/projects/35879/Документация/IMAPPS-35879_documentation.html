IMAPPS-35879. Доработки по задаче 35879

Конфигурация: база Wim_Du (исходники: C:\1c\Cursor_1c\WORK\Wim_Du).
Объект: Документ Сделка, модуль объекта.
Замена в расширении: функция ТекстЗапросаТаблицыПоПродажамБУ_БезНКД(НомераТаблиц).

Цель доработок: оптимизация пакетного запроса при проведении документа Сделка (БУ, без НКД) и корректное получение дат документов-партий в запросе. Задача возникла в рамках нагрузочного тестирования (исполнение большого числа сделок); узкое место — выполнение пакета запросов в модуле объекта Сделка.
Назначение расширения: расширение optOst реализовано для демонстрации изменений. Вендору базовой конфигурации требуется перенести все правки, помеченные комментарием // IMAPPS-35879, в базовую конфигурацию; после этого доработки будут действовать без расширения.

1. Базовая конфигурация: заменяемая функция

В базовой конфигурации в модуле объекта документа Сделка экспортная функция ТекстЗапросаТаблицыПоПродажамБУ_БезНКД(НомераТаблиц) формирует текст пакетного запроса для получения таблиц по продажам БУ (без НКД). Запрос используется при проведении сделки и заполняет временные таблицы, в том числе:

  • ВТ_ХозрасчетныйОстатки — остатки по регистру бухгалтерии Хозрасчетный (счета остатков, договор ДУ, актив, валюта, субконто);
  • ВТ_ОстаткиТело, ВТ_ОстаткиПереоценка, ТаблицаАктивы и др.

Первый блок запроса заполняет ВТ_ХозрасчетныйОстатки. В базовой конфигурации для этого используется виртуальная таблица РегистрБухгалтерии.Хозрасчетный.Остатки(&Период, ...) при &Период = МоментВремени(). При неудобной границе периода платформа может обрабатывать расширенный диапазон движений, что увеличивает время выполнения при массовом проведении.

Кроме того, в запросе используются обращения к полю даты через обобщённую ссылку (Субконто2.Дата, Партия.Дата). Для полиморфных ссылок (разные типы документов в Субконто2) в базе мог использоваться вариант ЕСТЬNULL(ВТ_ХозрасчетныйОстатки.Субконто2.Дата, ...), что не всегда корректно раскрывается платформой для полиморфного поля.

2. Что сделано в расширении (помечено IMAPPS-35879)

2.1. Выбор фрагмента запроса ВТ_ХозрасчетныйОстатки по месяцу документа

ГдеЧто сделано
Начало тела функции (после вставок в НомераТаблиц) Введён признак МесяцДокументаСовпадаетСТекущим = (НачалоМесяца(Дата) = НачалоМесяца(ТекущаяДатаСеанса())). По нему выбирается один из двух вариантов фрагмента запроса, заполняющего ВТ_ХозрасчетныйОстатки.
Формирование текста запроса Текст первого блока собирается из результата функции ПолучитьТекстФрагментаВТ_ХозрасчетныйОстатки(МесяцДокументаСовпадаетСТекущим) и общего хвоста (ИНДЕКСИРОВАТЬ ПО Счет и далее вплоть до блока ВТ_ОстаткиТело и остального запроса).
Новая функция в модуле объекта ПолучитьТекстФрагментаВТ_ХозрасчетныйОстатки(МесяцДокументаСовпадаетСТекущим) возвращает только фрагмент до «ИНДЕКСИРОВАТЬ ПО Счет»:
• при Истина — вариант с актуальными остатками (Остатки с пустым периодом) и оборотами за &Период с минусом, с группировкой (оптимизированный вариант для текущего месяца);
• при Ложь — вариант с одним вызовом Остатки(&Период, ...) (как в базе).
Логика обоих вариантов эквивалентна: результат — остатки на момент &Период; разница только в способе вычисления и в производительности.

2.2. Корректное получение даты документа по полиморфной ссылке (Субконто2, Партия)

В запросе три участка, где по полиморфной ссылке (Субконто2 или Партия) нужна дата документа. В базовой конфигурации мог использоваться доступ к полю даты через обобщённую ссылку. В доработке это заменено на явный ВЫБОР КОГДА ТИПЗНАЧЕНИЯ(...) = ТИП(Документ.XXX) ТОГДА ВЫРАЗИТЬ(... КАК Документ.XXX).Дата ... ИНАЧЕ ДАТАВРЕМЯ(1,1,1,0,0,0) КОНЕЦ по всем типам документов, которые могут находиться в Субконто2/Партии.

Участок запросаЧто заменено (IMAPPS-35879)
Список полей «ОстаткиАктивов»: поле ДокументОприходованияДата Вместо ЕСТЬNULL(ВТ_ХозрасчетныйОстатки.Субконто2.Дата, ДАТАВРЕМЯ(1,1,1,0,0,0)) — ВЫБОР по типам: Сделка, ПогашениеОблигаций, ДокументРасчетовСКонтрагентом, СделкаРЕПОПерваяНога, ВводВыводЦенныхБумагИзДУ, ВводНачальныхОстатовПоНКД, СделкаРЕПОВтораяНога, ПринятиеКУчетуАктивов, СделкаНаСрочномРынке.
СГРУППИРОВАТЬ ПО того же блока Аналогично: вместо группировки по ЕСТЬNULL(ВТ_ХозрасчетныйОстатки.Субконто2.Дата, ...) — группировка по тому же ВЫБОР по типам документов.
Поле Дата в блоке с ВТ_ОстаткиТело и ВТ_ОстаткиПереоценка Вместо ЕСТЬNULL(ВТ_ОстаткиТело.Партия.Дата, ВТ_ОстаткиПереоценка.Партия.Дата) — ВЫБОР по типам документов для ЕСТЬNULL(ВТ_ОстаткиТело.Партия, ВТ_ОстаткиПереоценка.Партия).

3. Способ реализации в расширении

  • Расширение конфигурации optOst (проект 35879, база Wim_Du).
  • В модуле объекта документа Сделка использована подмена метода: &Вместо("ТекстЗапросаТаблицыПоПродажамБУ_БезНКД"), экспортная функция optOst_ТекстЗапросаТаблицыПоПродажамБУ_БезНКД(НомераТаблиц).
  • Вся логика формирования текста запроса сосредоточена в подменённой функции и во вспомогательной функции ПолучитьТекстФрагментаВТ_ХозрасчетныйОстатки; общий хвост запроса (после фрагмента ВТ_ХозрасчетныйОстатки) объединён в один многострочный литерал и склеивается с результатом плейсхолдера.
  • Все изменённые фрагменты в коде помечены парой комментариев // IMAPPS-35879 ++ и // IMAPPS-35879 -- для удобного поиска при переносе в базовую конфигурацию.

4. Что переносить в базовую конфигурацию (для вендора)

  1. В модуле объекта документа Сделка в функции ТекстЗапросаТаблицыПоПродажамБУ_БезНКД(НомераТаблиц):
    • добавить вычисление МесяцДокументаСовпадаетСТекущим и сборку текста первого блока через плейсхолдер (или встроить оба варианта фрагмента в саму функцию без отдельной процедуры — по решению вендора);
    • заменить три участка с обращением к дате по полиморфной ссылке на ВЫБОР по типам документов, как в расширении.
  2. Добавить в тот же модуль объекта функцию ПолучитьТекстФрагментаВТ_ХозрасчетныйОстатки(МесяцДокументаСовпадаетСТекущим), если в базе решено выносить фрагменты в отдельную функцию (в расширении она помечена IMAPPS-35879 от начала до конца).

Расширение optOst можно использовать как эталон при переносе: все места с метками // IMAPPS-35879 в файле Расширения/optOst/Documents/Сделка/Ext/ObjectModule.bsl соответствуют требуемым доработкам.

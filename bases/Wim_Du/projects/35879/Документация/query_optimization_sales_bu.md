# Query optimization for sales BU

## Контекст

- Задача: `IMAPPS-35879` (нагрузочное тестирование исполнения 100000 сделок).
- По файлу замеров узкое место: `Документ.Сделка.МодульОбъекта`, строка `Результат = Запрос.ВыполнитьПакет();`.
- Поиск по исходникам базовой конфигурации (`C:\1c\Cursor_1c\WORK\Wim_Du`) показал точку:
  - `SRC/CF/Documents/Сделка/Ext/ObjectModule.bsl`
  - функция `ТекстЗапросаТаблицыПоПродажамБУ_БезНКД(НомераТаблиц)`
  - первый блок формирует `ВТ_ХозрасчетныйОстатки` через
    `РегистрБухгалтерии.Хозрасчетный.Остатки(&Период, ...)`
    при `&Период = МоментВремени()`.

## Наблюдение по механике виртуальной таблицы Остатки

- Виртуальная таблица `Остатки` рассчитывает результат через таблицу итогов и корректировку движениями.
- При неудобной границе периода платформа может тянуть расширенный диапазон движений, что удорожает запрос.
- Для регистра бухгалтерии это особенно заметно на больших объемах в момент пакетного запроса.

Использованные материалы:

- [Open Budget: виртуальные таблицы Остатки](https://open-budget.ru/public/1064528/)
- [1С ITS: эффективное обращение к виртуальной таблице Остатки](https://its.1c.ru/db/v8std/content/733/hdoc)

## Предложенная оптимизация

Идея:

1. Получить остатки на `КонецМесяца` (`&ПериодКонецМесяца`) из `Остатки(...)`.
2. Отдельно получить обороты за хвост периода `(&Период, &ПериодКонецМесяца]` из `Обороты(...)`.
3. Рассчитать остаток на `&Период` формулой:
   - `ОстатокНаПериод = ОстатокНаКонецМесяца - ОборотХвоста`.

Почему это должно ускорить:

- чтение остатков на конец месяца чаще опирается на реальные итоговые данные;
- при проведении документа в конце очереди обороты хвоста минимальны;
- суммарно снижается объем тяжелой части вычислений в первом блоке пакета.

## Что подготовлено в проекте

- Добавлен отдельный BSL-файл:
  - `Расширения/sales_bu_query_optimization.bsl`
- В файле:
  - `ТекстЗапросаВТХозрасчетныйОстаткиОптимизированный()` - текст оптимизированного первого блока;
  - `УстановитьПараметрыОптимизированногоЗапроса(Запрос, ПериодДокумента)` - установка `Период` и `ПериодКонецМесяца`.

## План встраивания в конфигурацию

1. В `Документ.Сделка.МодульОбъекта` заменить первый блок текста запроса на вызов функции из вынесенного модуля.
2. Добавить параметр `ПериодКонецМесяца`:
   - `КонецДня(КонецМесяца(МоментВремени()))`.
3. Оставшиеся блоки пакета не менять (минимальный риск регрессии).

## Проверка корректности

Проверки после встраивания:

1. Сравнить выходные таблицы старого и нового запроса на одинаковом наборе сделок.
2. Проверить граничные случаи:
   - переход месяца;
   - отсутствие оборотов хвоста;
   - отрицательные остатки/переоценка.
3. Повторить замер на 100000 сделках и сравнить время только для `ТаблицыПоПродажам_БУ`.

## Ограничения и риски

- Нужно подтвердить знак оборотов (`Дт/Кт`) на боевом наборе тест-кейсов, чтобы исключить инверсию.
- Если документ не всегда последний в хронологии, объем хвоста вырастет и эффект может снизиться.
- Требуется отдельная валидация по НУ/валютным полям для всех типов активов.
